Урок 1: Введение в безопасность Python

Почему безопасность важна?

Безопасность — это не функция, а свойство системы. Уязвимости могут привести к:

  • Утечке конфиденциальных данных пользователей
  • Финансовым потерям
  • Потере репутации и доверия
  • Юридическим последствиям (GDPR, 152-ФЗ)
  • Полной компрометации системы
Важно: 84% кибератак используют уязвимости на уровне приложений. Безопасность нужно встраивать на всех этапах разработки.

Урок 2: SQL-инъекции

Что такое SQL-инъекция?

SQL-инъекция (SQLi) — это атака, при которой злоумышленник внедряет вредоносный SQL-код в запрос, чтобы получить несанкционированный доступ к базе данных.

❌ Уязвимый код:

# НИКОГДА ТАК НЕ ДЕЛАЙТЕ!
import sqlite3

def get_user(username, password):
    conn = sqlite3.connect('users.db')
    cursor = conn.cursor()
    
    # Прямая подстановка в запрос — опасно!
    query = f"SELECT * FROM users WHERE username = '{username}' AND password = '{password}'"
    cursor.execute(query)
    return cursor.fetchone()

# Пример атаки:
# username = "admin' --"
# password = "anything"
# Запрос станет: SELECT * FROM users WHERE username = 'admin' --' AND password = 'anything'
# -- это комментарий, условие на пароль игнорируется!

✅ Безопасный код с параметризацией:

import sqlite3

def get_user_safe(username, password):
    conn = sqlite3.connect('users.db')
    cursor = conn.cursor()
    
    # Параметризованный запрос
    query = "SELECT * FROM users WHERE username = ? AND password = ?"
    cursor.execute(query, (username, password))
    return cursor.fetchone()

# С параметризацией SQL-инъекция невозможна!

Защита от SQL-инъекций

1. Использование ORM (SQLAlchemy, Django ORM):

# SQLAlchemy
from sqlalchemy import text

def get_user_orm(session, username):
    # ORM автоматически экранирует параметры
    user = session.query(User).filter(User.username == username).first()
    return user

# Или с сырым SQL через text()
result = session.execute(
    text("SELECT * FROM users WHERE username = :username"),
    {"username": username}
)

2. Экранирование вручную (если нет ORM):

import mysql.connector

def get_user_escaped(username):
    conn = mysql.connector.connect(...)
    cursor = conn.cursor(prepared=True)  # Подготовленные запросы
    
    cursor.execute("SELECT * FROM users WHERE username = %s", (username,))
    return cursor.fetchone()

3. Ограничение привилегий БД:

# Создаем отдельного пользователя для приложения с минимальными правами
GRANT SELECT, INSERT, UPDATE ON app_db.* TO 'app_user'@'localhost';
# Без DROP, ALTER, DELETE (если не нужны)

Практический совет: Используйте ORM (SQLAlchemy) или подготовленные запросы. Никогда не конкатенируйте строки при построении SQL-запросов. Даже если кажется, что данные безопасны.

Урок 3: XSS и CSRF атаки

XSS (Cross-Site Scripting)

XSS — атака, при которой злоумышленник внедряет вредоносный JavaScript-код на страницу, который выполняется в браузере жертвы.

❌ Уязвимый код (Flask без экранирования):

from flask import Flask, request, render_template_string

app = Flask(__name__)

@app.route('/comment')
def comment():
    # Пользовательский ввод выводится без экранирования
    user_comment = request.args.get('text', '')
    return render_template_string(f"
{user_comment}
")

# Пример атаки:
# /comment?text=

✅ Безопасный код с экранированием:

from flask import Flask, request, render_template_string
from markupsafe import escape

@app.route('/comment')
def comment_safe():
    user_comment = request.args.get('text', '')
    # escape() экранирует HTML-теги
    safe_comment = escape(user_comment)
    return render_template_string(f"
{safe_comment}
")

# Или в Jinja2 шаблонах по умолчанию всё экранируется:
# {{ user_comment }} — автоматическое экранирование
# {{ user_comment|safe }} — отключает экранирование (используйте осторожно!)

Типы XSS:

  • Reflected XSS: Код отражается в ответе (через GET параметры)
  • Stored XSS: Код сохраняется в БД и выполняется при просмотре
  • DOM-based XSS: Уязвимость в клиентском JavaScript

CSRF (Cross-Site Request Forgery)

CSRF — атака, при которой злоумышленник заставляет жертву выполнить нежелательное действие на сайте, где она аутентифицирована.

# Пример CSRF атаки
# Злоумышленник размещает на своем сайте:
# 

# Если пользователь авторизован в банке, произойдет перевод

# Защита: CSRF-токены
from flask_wtf.csrf import CSRFProtect

app = Flask(__name__)
csrf = CSRFProtect(app)

# В формах автоматически добавляется токен
# В AJAX запросах нужно передавать токен в заголовках
# X-CSRFToken: token-value

Защита от CSRF:

  • CSRF-токены в формах
  • Проверка заголовка Referer / Origin
  • SameSite cookies (SameSite=Lax или Strict)
  • Использование CAPTCHA для критических действий
# Настройка SameSite cookies в Flask
app.config.update(
    SESSION_COOKIE_SAMESITE='Lax',
    SESSION_COOKIE_SECURE=True,  # только HTTPS
    SESSION_COOKIE_HTTPONLY=True  # недоступны из JS
)

Правила защиты:

  • Всегда экранируйте вывод пользовательских данных
  • Используйте Content-Security-Policy (CSP) заголовки
  • Включайте CSRF-защиту для всех изменяющих запросов
  • Устанавливайте Secure и HttpOnly флаги для cookies

Урок 4: Небезопасная десериализация

Что такое десериализация?

Десериализация — это процесс восстановления объекта из его сериализованного представления (JSON, pickle, XML). Небезопасная десериализация может привести к выполнению произвольного кода.

Критическая опасность! Десериализация из ненадежных источников может привести к RCE (Remote Code Execution).

❌ Опасный код с pickle:

import pickle

def load_user_data(data):
    # НИКОГДА так не делайте!
    user_data = pickle.loads(data)  # Злоумышленник может выполнить произвольный код
    return user_data

# Злоумышленник может отправить:
# __import__('os').system('rm -rf /')
# Это уничтожит систему при десериализации!

✅ Безопасные альтернативы:

import json
import ast

# Используйте JSON для обмена данными
def load_user_data_safe(data):
    try:
        user_data = json.loads(data)  # JSON безопасен для десериализации
        return user_data
    except json.JSONDecodeError:
        return None

# Или ast.literal_eval для Python-литералов
def safe_eval(data):
    # ast.literal_eval работает только с литералами (строки, числа, списки, словари)
    return ast.literal_eval(data)

Сравнение форматов:

Формат Безопасность Когда использовать
JSON Безопасен API, обмен данными, конфиги
pickle ОПАСЕН Только для доверенных данных (внутри приложения)
yaml Опасен Используйте yaml.safe_load()
xml Опасен Используйте defusedxml

Безопасная десериализация в реальных проектах

# Использование defusedxml для защиты от XML-атак (XXE, Billion Laughs)
import defusedxml.ElementTree as ET

def parse_xml_safe(xml_data):
    # defusedxml защищает от XML-бомб и внешних сущностей
    tree = ET.fromstring(xml_data)
    return tree

# Безопасный YAML
import yaml

def load_yaml_safe(yaml_data):
    # Всегда используйте safe_load!
    return yaml.safe_load(yaml_data)  # Не позволяет создавать произвольные объекты

Золотое правило: Никогда не десериализуйте данные из ненадежных источников. Если это необходимо, используйте безопасные форматы (JSON) и валидацию схемы.

Урок 5: Безопасная аутентификация

Хранение паролей

Никогда не храните пароли в открытом виде! Используйте современные алгоритмы хеширования.

import bcrypt
import hashlib

# ❌ НИКОГДА ТАК НЕ ДЕЛАЙТЕ!
def store_password_insecure(password):
    hashed = hashlib.md5(password.encode()).hexdigest()  # MD5 небезопасен!
    return hashed

# ✅ Правильное хранение паролей с bcrypt
import bcrypt

def hash_password(password):
    # Генерируем соль и хеш
    salt = bcrypt.gensalt(rounds=12)  # 12 раундов — достаточно
    hashed = bcrypt.hashpw(password.encode('utf-8'), salt)
    return hashed

def verify_password(password, hashed):
    return bcrypt.checkpw(password.encode('utf-8'), hashed)

# ✅ Альтернатива — argon2 (более современный)
from argon2 import PasswordHasher

ph = PasswordHasher()

def hash_password_argon2(password):
    return ph.hash(password)

def verify_password_argon2(password, hashed):
    try:
        ph.verify(hashed, password)
        return True
    except:
        return False

JWT (JSON Web Tokens)

JWT — популярный формат для аутентификации. Но важно использовать его правильно!

import jwt
from datetime import datetime, timedelta

SECRET_KEY = "your-secret-key-here"  # Должен быть в переменных окружения!

def create_token(user_id):
    payload = {
        'user_id': user_id,
        'exp': datetime.utcnow() + timedelta(hours=24),  # Время жизни токена
        'iat': datetime.utcnow(),  # Время выдачи
        'iss': 'pymaster.com'  # Издатель
    }
    token = jwt.encode(payload, SECRET_KEY, algorithm='HS256')
    return token

def verify_token(token):
    try:
        # Проверяем подпись и срок действия
        payload = jwt.decode(token, SECRET_KEY, algorithms=['HS256'])
        return payload
    except jwt.ExpiredSignatureError:
        return None  # Токен истек
    except jwt.InvalidTokenError:
        return None  # Невалидный токен

# ⚠️ Важно! Не используйте algorithm='none' и не проверяйте подпись!

Безопасность JWT:

  • Используйте короткое время жизни (15-60 минут) + refresh tokens
  • Храните секретный ключ в переменных окружения
  • Используйте только HS256 или RS256 алгоритмы
  • Не храните чувствительные данные в payload
  • Храните токены в httpOnly cookies, а не в localStorage

Урок 7: Практика — сканер уязвимостей

Проект: Простой сканер SQL-инъекций

Создадим простой сканер для обнаружения SQL-инъекций в веб-приложениях (для тестирования собственных проектов!).

import requests
import time
from urllib.parse import urljoin

class SQLInjectionScanner:
    def __init__(self, base_url):
        self.base_url = base_url
        self.payloads = [
            "' OR '1'='1",
            "' OR 1=1--",
            "' UNION SELECT NULL--",
            "'; DROP TABLE users--",
            "' AND SLEEP(5)--",
        ]
        self.timeout = 5
        
    def test_sql_injection(self, url, param):
        """Тестирует параметр на наличие SQL-инъекции"""
        results = []
        
        for payload in self.payloads:
            # Формируем тестовый запрос
            test_params = {param: payload}
            try:
                start_time = time.time()
                response = requests.get(url, params=test_params, timeout=self.timeout)
                response_time = time.time() - start_time
                
                # Проверка на задержку (time-based injection)
                if response_time > 4.5:  # Если ответ пришел через 4.5+ секунд
                    results.append({
                        'payload': payload,
                        'vulnerable': True,
                        'type': 'Time-based SQL injection',
                        'response_time': response_time
                    })
                # Проверка на ошибки БД
                elif 'sql' in response.text.lower() or 'mysql' in response.text.lower():
                    results.append({
                        'payload': payload,
                        'vulnerable': True,
                        'type': 'Error-based SQL injection',
                        'response_length': len(response.text)
                    })
                else:
                    results.append({
                        'payload': payload,
                        'vulnerable': False
                    })
                    
            except requests.exceptions.RequestException as e:
                results.append({
                    'payload': payload,
                    'vulnerable': False,
                    'error': str(e)
                })
                
        return results
    
    def scan_parameters(self, url, params):
        """Сканирует все параметры URL"""
        all_results = {}
        
        for param in params:
            print(f"Тестируем параметр: {param}")
            results = self.test_sql_injection(url, param)
            all_results[param] = results
            
            # Выводим результаты
            for result in results:
                if result.get('vulnerable'):
                    print(f"  🔴 НАЙДЕНО! {param} = {result['payload']}")
                    print(f"     Тип: {result.get('type')}")
                else:
                    print(f"  🟢 Безопасно: {result['payload']}")
                    
        return all_results

# Пример использования
if __name__ == "__main__":
    # Только для тестирования своих приложений!
    scanner = SQLInjectionScanner("http://localhost:5000")
    
    # URL и параметры для тестирования
    test_url = "http://localhost:5000/search"
    test_params = ['q', 'page', 'id']
    
    results = scanner.scan_parameters(test_url, test_params)
