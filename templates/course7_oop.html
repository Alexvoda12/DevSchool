Урок 1: Классы и объекты

Что такое класс и объект?

Класс — это чертеж, шаблон, по которому создаются объекты. Представьте, что класс — это форма для выпечки печенья, а объект — само печенье.

Объект — это конкретный экземпляр класса, обладающий своими значениями атрибутов.

# Определение класса
class Dog:
    # Атрибут класса (общий для всех объектов)
    species = "Canis familiaris"
    
    # Конструктор (инициализатор)
    def __init__(self, name, age):
        self.name = name  # Атрибут объекта
        self.age = age    # Атрибут объекта
    
    # Метод класса
    def bark(self):
        return f"{self.name} says Woof!"

# Создание объектов
my_dog = Dog("Buddy", 3)
your_dog = Dog("Lucy", 5)

print(my_dog.name)      # Buddy
print(my_dog.bark())    # Buddy says Woof!
print(my_dog.species)   # Canis familiaris
print(your_dog.age)     # 5

Метод __init__

__init__ — это специальный метод (конструктор), который автоматически вызывается при создании объекта. Здесь мы задаем начальные значения атрибутов.

class Car:
    def __init__(self, brand, model, year):
        self.brand = brand
        self.model = model
        self.year = year
        self.mileage = 0  # значение по умолчанию
    
    def drive(self, km):
        self.mileage += km
        return f"Пробег увеличен на {km} км"

my_car = Car("Toyota", "Camry", 2020)
print(my_car.mileage)  # 0
my_car.drive(100)
print(my_car.mileage)  # 100

Ключевое слово self:

self ссылается на текущий объект. Это первый параметр любого метода класса. Через self мы обращаемся к атрибутам и методам объекта.

Урок 2: Атрибуты и методы

Типы атрибутов

Атрибуты класса — общие для всех объектов.

Атрибуты объекта — уникальные для каждого экземпляра.

class Employee:
    # Атрибут класса
    company = "Tech Corp"
    employee_count = 0
    
    def __init__(self, name, salary):
        # Атрибуты объекта
        self.name = name
        self.salary = salary
        Employee.employee_count += 1
    
    def get_info(self):
        return f"{self.name} работает в {self.company}, зарплата: {self.salary}"

emp1 = Employee("Анна", 50000)
emp2 = Employee("Иван", 60000)

print(emp1.get_info())  # Анна работает в Tech Corp, зарплата: 50000
print(emp2.company)     # Tech Corp
print(Employee.employee_count)  # 2

Типы методов

1. Обычные методы — работают с атрибутами объекта (self)

class Calculator:
    def __init__(self, value):
        self.value = value
    
    def add(self, x):
        self.value += x
        return self.value

2. Методы класса (@classmethod) — работают с атрибутами класса (cls)

class Employee:
    bonus_rate = 1.1
    
    @classmethod
    def set_bonus_rate(cls, rate):
        cls.bonus_rate = rate
    
    @classmethod
    def from_string(cls, emp_string):
        name, salary = emp_string.split('-')
        return cls(name, int(salary))

3. Статические методы (@staticmethod) — не зависят от класса или объекта

class Math:
    @staticmethod
    def add(x, y):
        return x + y
    
    @staticmethod
    def is_even(num):
        return num % 2 == 0

print(Math.add(5, 3))        # 8
print(Math.is_even(4))       # True

Урок 3: Инкапсуляция

Сокрытие данных

Инкапсуляция — это механизм, который скрывает внутренние данные объекта и предоставляет контролируемый доступ к ним.

Уровни доступа в Python:

  • public — доступны везде (обычные атрибуты)
  • protected — одно подчеркивание _attr (соглашение, что это внутреннее использование)
  • private — два подчеркивания __attr (name mangling)
class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner           # public
        self._interest_rate = 0.05   # protected
        self.__balance = balance     # private
    
    # Геттер для приватного атрибута
    def get_balance(self):
        return self.__balance
    
    # Сеттер с проверкой
    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount
            return True
        return False
    
    def withdraw(self, amount):
        if 0 < amount <= self.__balance:
            self.__balance -= amount
            return True
        return False

account = BankAccount("Иван", 1000)
print(account.owner)           # Иван (public - доступно)
print(account._interest_rate)  # 0.05 (protected - доступно, но не рекомендуется)
# print(account.__balance)     # Ошибка! AttributeError

# Доступ через методы
print(account.get_balance())   # 1000
account.deposit(500)
print(account.get_balance())   # 1500

# Обход name mangling (не делайте так!)
print(account._BankAccount__balance)  # 1500 (технически доступно, но нарушает инкапсуляцию)

Свойства (@property)

Декоратор property позволяет создавать геттеры, сеттеры и делитеры для удобного доступа к приватным атрибутам.

class Person:
    def __init__(self, name, age):
        self._name = name
        self._age = age
    
    @property
    def age(self):
        """Геттер для возраста"""
        return self._age
    
    @age.setter
    def age(self, value):
        """Сеттер с валидацией"""
        if value < 0:
            raise ValueError("Возраст не может быть отрицательным")
        if value > 150:
            raise ValueError("Слишком большой возраст")
        self._age = value
    
    @property
    def name(self):
        return self._name
    
    @name.deleter
    def name(self):
        print("Удаляем имя...")
        del self._name

p = Person("Анна", 25)
print(p.age)      # 25 (используем как атрибут)
p.age = 26        # Работает сеттер
# p.age = -5      # ValueError!
# del p.name      # Удаляет атрибут

Урок 4: Наследование

Базовое наследование

Наследование позволяет создавать новый класс на основе существующего. Дочерний класс получает все атрибуты и методы родителя и может их расширять или изменять.

class Animal:
    def __init__(self, name):
        self.name = name
    
    def speak(self):
        return f"{self.name} издает звук"
    
    def move(self):
        return f"{self.name} двигается"

# Наследование
class Dog(Animal):
    def speak(self):  # Переопределение метода
        return f"{self.name} лает: Гав-гав!"
    
    def fetch(self):  # Новый метод
        return f"{self.name} приносит палку"

class Cat(Animal):
    def speak(self):
        return f"{self.name} мяукает: Мяу-мяу!"

# Создание объектов
dog = Dog("Бобик")
cat = Cat("Мурка")

print(dog.speak())  # Бобик лает: Гав-гав!
print(cat.speak())  # Мурка мяукает: Мяу-мяу!
print(dog.move())   # Бобик двигается (унаследовано)
print(dog.fetch())  # Бобик приносит палку

Функции super()

super() позволяет вызвать метод родительского класса. Это полезно, когда мы хотим расширить родительский метод, а не полностью заменить его.

class Vehicle:
    def __init__(self, brand, model):
        self.brand = brand
        self.model = model
        self.speed = 0
    
    def info(self):
        return f"Транспорт: {self.brand} {self.model}"

class Car(Vehicle):
    def __init__(self, brand, model, doors):
        # Вызываем конструктор родителя
        super().__init__(brand, model)
        self.doors = doors
    
    def info(self):
        # Расширяем родительский метод
        base_info = super().info()
        return f"{base_info}, дверей: {self.doors}"
    
    def accelerate(self):
        self.speed += 10
        return f"Скорость: {self.speed} км/ч"

my_car = Car("Toyota", "Camry", 4)
print(my_car.info())  # Транспорт: Toyota Camry, дверей: 4

Множественное наследование

Python поддерживает множественное наследование — класс может наследоваться от нескольких родителей.

class Flyer:
    def fly(self):
        return "Я могу летать"

class Swimmer:
    def swim(self):
        return "Я могу плавать"

class Duck(Flyer, Swimmer):
    def __init__(self, name):
        self.name = name
    
    def speak(self):
        return f"{self.name} крякает"

duck = Duck("Дональд")
print(duck.fly())   # Я могу летать
print(duck.swim())  # Я могу плавать
print(duck.speak()) # Дональд крякает

Важно! При множественном наследовании порядок классов важен. Python ищет методы слева направо (MRO - Method Resolution Order).

Урок 5: Полиморфизм

Что такое полиморфизм?

Полиморфизм — это способность объектов разных классов отвечать на одни и те же методы по-разному. Один интерфейс — разная реализация.

class Shape:
    def area(self):
        pass
    
    def perimeter(self):
        pass

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height
    
    def area(self):
        return self.width * self.height
    
    def perimeter(self):
        return 2 * (self.width + self.height)

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
    
    def area(self):
        return 3.14159 * self.radius ** 2
    
    def perimeter(self):
        return 2 * 3.14159 * self.radius

class Triangle(Shape):
    def __init__(self, a, b, c):
        self.a = a
        self.b = b
        self.c = c
    
    def area(self):
        # Формула Герона
        s = (self.a + self.b + self.c) / 2
        return (s * (s - self.a) * (s - self.b) * (s - self.c)) ** 0.5
    
    def perimeter(self):
        return self.a + self.b + self.c

# Полиморфизм в действии
shapes = [
    Rectangle(5, 3),
    Circle(4),
    Triangle(3, 4, 5)
]

for shape in shapes:
    print(f"Площадь: {shape.area():.2f}")
    print(f"Периметр: {shape.perimeter():.2f}")
    print("---")

Duck Typing

В Python полиморфизм основан на принципе "утиной типизации": "Если это ходит как утка и крякает как утка, значит это утка". Важно не то, к какому классу принадлежит объект, а какие методы он поддерживает.

class Duck:
    def speak(self):
        return "Кря-кря"
    
    def walk(self):
        return "Утка идет вразвалочку"

class Dog:
    def speak(self):
        return "Гав-гав"
    
    def walk(self):
        return "Собака бежит"

class Cat:
    def speak(self):
        return "Мяу-мяу"
    
    def walk(self):
        return "Кошка крадется"

def animal_actions(animal):
    # Не важно, какого класса animal
    # Главное, чтобы у него были методы speak и walk
    print(animal.speak())
    print(animal.walk())

# Один и тот же код работает с разными объектами
animal_actions(Duck())
animal_actions(Dog())
animal_actions(Cat())

Урок 6: Магические методы

Что такое магические методы?

Магические методы (dunder methods) — это специальные методы с двойными подчеркиваниями, которые определяют поведение объектов в разных ситуациях.

Основные магические методы:

__init__(self, ...) — конструктор, вызывается при создании объекта
__str__(self) — для пользователей (print, str)
__repr__(self) — для разработчиков (repr, отладка)
__len__(self) — для функции len()
__add__(self, other) — оператор +
__eq__(self, other) — оператор ==
__lt__(self, other) — оператор <
__getitem__(self, key) — для доступа по индексу obj[key]
__call__(self, ...) — делает объект вызываемым (как функцию) 
class Book:
    def __init__(self, title, author, pages):
        self.title = title
        self.author = author
        self.pages = pages
        self.current_page = 0
    
    def __str__(self):
        return f"'{self.title}' by {self.author}"
    
    def __repr__(self):
        return f"Book('{self.title}', '{self.author}', {self.pages})"
    
    def __len__(self):
        return self.pages
    
    def __add__(self, other):
        """Объединение двух книг"""
        new_title = f"{self.title} & {other.title}"
        new_author = f"{self.author}, {other.author}"
        new_pages = self.pages + other.pages
        return Book(new_title, new_author, new_pages)
    
    def __eq__(self, other):
        return self.title == other.title and self.author == other.author
    
    def __getitem__(self, page_num):
        if 0 <= page_num < self.pages:
            return f"Страница {page_num + 1} книги '{self.title}'"
        raise IndexError("Страница не существует")
    
    def __call__(self, page_num):
        """Перелистывание на страницу"""
        self.current_page = page_num
        return f"Открыта страница {page_num}"

# Использование
book1 = Book("1984", "Джордж Оруэлл", 328)
book2 = Book("Скотный двор", "Джордж Оруэлл", 112)

print(str(book1))           # '1984' by Джордж Оруэлл
print(repr(book1))          # Book('1984', 'Джордж Оруэлл', 328)
print(len(book1))           # 328

book3 = book1 + book2
print(book3)                # '1984 & Скотный двор' by Джордж Оруэлл, Джордж Оруэлл

print(book1 == book2)       # False

print(book1[100])           # Страница 101 книги '1984'
print(book1(50))            # Открыта страница 50
