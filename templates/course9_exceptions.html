Урок 1: Что такое исключения?

Понятие исключения

Исключения (Exceptions) — это ошибки, возникающие во время выполнения программы. В отличие от синтаксических ошибок, которые Python обнаруживает до запуска, исключения возникают уже в процессе работы.

Если исключение не обработать, программа аварийно завершится. Но мы можем "поймать" исключение и выполнить другой код вместо падения.

Популярные типы исключений:

Исключение Когда возникает
ZeroDivisionError Деление на ноль
NameError Переменная не найдена
TypeError Операция над объектом неподходящего типа
ValueError Функция получает аргумент правильного типа, но с некорректным значением
IndexError Индекс вне границ списка
KeyError Ключ отсутствует в словаре
FileNotFoundError Попытка открыть несуществующий файл 
# Примеры исключений

# ZeroDivisionError
# print(10 / 0)  # Ошибка: деление на ноль

# NameError
# print(unknown_variable)  # Ошибка: переменная не определена

# TypeError
# print("5" + 5)  # Ошибка: нельзя сложить строку и число

# ValueError
# int("abc")  # Ошибка: нельзя преобразовать "abc" в число

# IndexError
my_list = [1, 2, 3]
# print(my_list[10])  # Ошибка: индекс вне диапазона

# KeyError
my_dict = {"a": 1, "b": 2}
# print(my_dict["c"])  # Ошибка: ключ "c" отсутствует

# FileNotFoundError

Урок 2: Конструкция try-except

Базовый синтаксис

Конструкция try...except позволяет "поймать" исключение и выполнить другой код вместо аварийного завершения.

try:
    # Код, который может вызвать исключение
    number = int(input("Введите число: "))
    result = 10 / number
    print(f"Результат: {result}")
except:
    # Код, который выполнится при возникновении любого исключения
    print("Произошла ошибка!")

print("Программа продолжает работу")

Обработка конкретных исключений:

try:
    number = int(input("Введите число: "))
    result = 10 / number
    print(f"Результат: {result}")
except ZeroDivisionError:
    print("Ошибка: деление на ноль!")
except ValueError:
    print("Ошибка: введено не число!")
except Exception as e:
    print(f"Произошла неизвестная ошибка: {e}")

Полная конструкция: try-except-else-finally

  • try: Код, который может вызвать исключение
  • except: Код, выполняемый при возникновении исключения
  • else: Код, выполняемый, если исключения НЕ было
  • finally: Код, выполняемый ВСЕГДА (было исключение или нет)
def divide_numbers(a, b):
    try:
        result = a / b
    except ZeroDivisionError:
        print("Ошибка: деление на ноль!")
        return None
    except TypeError:
        print("Ошибка: аргументы должны быть числами!")
        return None
    else:
        print("Деление выполнено успешно!")
        return result
    finally:
        print("Блок finally выполняется всегда")

# Тестируем
print(divide_numbers(10, 2))   # Успех
print(divide_numbers(10, 0))   # Деление на ноль
print(divide_numbers(10, "2")) # Ошибка типа

Важно! Не ловите все исключения подряд без разбора (except:). Это может скрыть серьезные ошибки и затруднить отладку. Всегда старайтесь указывать конкретные типы исключений.

Урок 3: Генерация исключений (raise)

Ключевое слово raise

Иногда нужно самим создать исключение — например, при проверке входных данных. Для этого используется raise.

def withdraw_money(balance, amount):
    if amount < 0:
        raise ValueError("Сумма снятия не может быть отрицательной!")
    if amount > balance:
        raise ValueError("Недостаточно средств на счете!")
    
    balance -= amount
    return balance

# Использование
try:
    new_balance = withdraw_money(1000, -500)
except ValueError as e:
    print(f"Ошибка: {e}")

Проброс исключений

Можно поймать исключение, частично обработать и пробросить дальше.

def safe_divide(a, b):
    try:
        result = a / b
    except ZeroDivisionError:
        print("Ловим деление на ноль, логируем...")
        # Пробрасываем исключение дальше
        raise
    return result

try:
    safe_divide(10, 0)
except ZeroDivisionError:
    print("Исключение проброшено и поймано здесь")

Урок 4: Создание своих исключений

Пользовательские исключения

Можно создавать собственные классы исключений, наследуясь от Exception. Это помогает точнее классифицировать ошибки в вашей программе.

# Создаем свое исключение
class InsufficientFundsError(Exception):
    """Исключение при недостатке средств"""
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(f"Недостаточно средств: запрошено {amount}, доступно {balance}")

class NegativeAmountError(Exception):
    """Исключение при отрицательной сумме"""
    pass

class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner
        self.balance = balance
    
    def withdraw(self, amount):
        if amount < 0:
            raise NegativeAmountError("Сумма снятия не может быть отрицательной")
        if amount > self.balance:
            raise InsufficientFundsError(self.balance, amount)
        
        self.balance -= amount
        return self.balance

# Использование
account = BankAccount("Иван", 1000)

try:
    account.withdraw(1500)
except InsufficientFundsError as e:
    print(f"Ошибка: {e}")
except NegativeAmountError:
    print("Сумма должна быть положительной")

Совет: Создавайте иерархию своих исключений. Например, базовое исключение BankError, а от него уже конкретные: InsufficientFundsError, InvalidTransactionError и т.д.
