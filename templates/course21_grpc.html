Урок 1: Введение в микросервисы

Монолит против микросервисов

Традиционные приложения часто строятся как монолит — одно большое приложение, содержащее весь функционал. С ростом сложности монолит становится тяжело масштабировать и поддерживать.

Микросервисная архитектура — это подход, при котором приложение разбивается на небольшие, независимые сервисы, каждый из которых отвечает за свою бизнес-функцию. Сервисы общаются между собой по сети.

Сравнение архитектур:

Характеристика Монолит Микросервисы
Масштабирование Все приложение целиком Каждый сервис независимо
Разработка Одна кодовая база Разные команды, разные технологии
Развертывание Единое развертывание Независимое развертывание сервисов
Технологии Один стек технологий Разные стеки под разные задачи
Сложность Низкая на старте Высокая (сетевое взаимодействие)

Способы коммуникации между сервисами

  • HTTP/REST: Самый распространенный, но не самый эффективный
  • gRPC: Высокопроизводительный RPC-фреймворк от Google
  • Message Brokers (RabbitMQ, Kafka): Асинхронная коммуникация через очереди
  • GraphQL: Гибкий запрос данных от клиента

В этом курсе мы сосредоточимся на gRPC — современном и производительном протоколе для микросервисов.

Урок 2: Что такое gRPC и Protocol Buffers

gRPC (gRPC Remote Procedure Call)

gRPC — это высокопроизводительный, открытый фреймворк для удаленного вызова процедур (RPC), разработанный Google. Он использует HTTP/2 для транспорта и Protocol Buffers для сериализации данных.

Ключевые особенности gRPC:

  • Высокая производительность: Бинарная сериализация Protobuf
  • Мультиязычность: Поддерживает Python, Go, Java, C++, Node.js и многие другие
  • Стриминг: Поддерживает односторонний и двусторонний стриминг
  • HTTP/2: Мультиплексирование запросов, сжатие заголовков
  • Автоматическая генерация кода: Клиенты и серверы генерируются из .proto файлов

Protocol Buffers (protobuf)

Protocol Buffers — это язык описания структур данных и механизм сериализации, разработанный Google. Он позволяет описывать сообщения в компактном бинарном формате.

# Пример .proto файла
syntax = "proto3";

// Сервис для работы с пользователями
service UserService {
    rpc GetUser (UserRequest) returns (UserResponse);
    rpc CreateUser (CreateUserRequest) returns (UserResponse);
    rpc ListUsers (Empty) returns (stream UserResponse); // Стриминг
}

// Сообщение для запроса
message UserRequest {
    int32 user_id = 1;
}

// Сообщение для ответа
message UserResponse {
    int32 id = 1;
    string name = 2;
    string email = 3;
    int32 age = 4;
}

message CreateUserRequest {
    string name = 1;
    string email = 2;
    int32 age = 3;
}

message Empty {}  // Пустое сообщение

Преимущества protobuf над JSON:

  • Размер: Бинарный формат занимает меньше места
  • Скорость: Сериализация/десериализация быстрее
  • Типизация: Строгая типизация полей
  • Обратная совместимость: Легко добавлять новые поля

Установка инструментов:

Для работы с gRPC в Python нужно установить:

pip install grpcio grpcio-tools protobuf

Шаг 2: Генерация Python кода

python -m grpc_tools.protoc -I. --python_out=. --grpc_python_out=. user_service.proto

Шаг 3: Реализация сервера

import grpc
from concurrent import futures
import user_service_pb2
import user_service_pb2_grpc

# База данных пользователей (в реальности - настоящая БД)
users_db = {
    1: {"id": 1, "name": "Анна", "email": "anna@example.com"},
    2: {"id": 2, "name": "Иван", "email": "ivan@example.com"},
}

class UserServiceServicer(user_service_pb2_grpc.UserServiceServicer):
    def GetUser(self, request, context):
        user_id = request.user_id
        user = users_db.get(user_id)
        
        if user:
            return user_service_pb2.UserResponse(
                id=user["id"],
                name=user["name"],
                email=user["email"]
            )
        else:
            context.set_code(grpc.StatusCode.NOT_FOUND)
            context.set_details("Пользователь не найден")
            return user_service_pb2.UserResponse()

def serve():
    server = grpc.server(futures.ThreadPoolExecutor(max_workers=10))
    user_service_pb2_grpc.add_UserServiceServicer_to_server(
        UserServiceServicer(), server
    )
    server.add_insecure_port('[::]:50051')
    server.start()
    print("gRPC сервер запущен на порту 50051")
    server.wait_for_termination()

if __name__ == '__main__':
    serve()

Урок 4: gRPC клиент на Python

Создание клиента

import grpc
import user_service_pb2
import user_service_pb2_grpc

def run():
    # Подключение к серверу
    with grpc.insecure_channel('localhost:50051') as channel:
        stub = user_service_pb2_grpc.UserServiceStub(channel)
        
        # Запрос пользователя с ID=1
        try:
            response = stub.GetUser(user_service_pb2.UserRequest(user_id=1))
            print(f"Пользователь найден: {response.name}, {response.email}")
        except grpc.RpcError as e:
            print(f"Ошибка: {e.code()} - {e.details()}")
        
        # Запрос несуществующего пользователя
        try:
            response = stub.GetUser(user_service_pb2.UserRequest(user_id=999))
            print(f"Пользователь: {response.name}")
        except grpc.RpcError as e:
            print(f"Ошибка: {e.code()} - {e.details()}")

if __name__ == '__main__':
    run()

Урок 5: Типы вызовов в gRPC

Четыре типа вызовов

1. Unary (запрос-ответ) - самый простой

# .proto
service Calculator {
    rpc Add (Numbers) returns (Result);
}

2. Server Streaming - сервер стримит данные

# .proto
service DataService {
    rpc GetUpdates (Empty) returns (stream Update);
}

# Сервер
def GetUpdates(self, request, context):
    for i in range(10):
        yield Update(message=f"Сообщение {i}")

# Клиент
for update in stub.GetUpdates(Empty()):
    print(update.message)

3. Client Streaming - клиент стримит данные

# .proto
service UploadService {
    rpc Upload (stream Chunk) returns (UploadResult);
}

# Сервер
def Upload(self, request_iterator, context):
    total_size = 0
    for chunk in request_iterator:
        total_size += len(chunk.data)
    return UploadResult(size=total_size)

# Клиент
def send_chunks(stub):
    def generate():
        for i in range(5):
            yield Chunk(data=f"Chunk {i}".encode())
    response = stub.Upload(generate())
    print(f"Загружено: {response.size} байт")

4. Bidirectional Streaming - двусторонний стриминг

# .proto
service ChatService {
    rpc Chat (stream ChatMessage) returns (stream ChatMessage);
}

# Сервер
def Chat(self, request_iterator, context):
    for message in request_iterator:
        # Обработка сообщения
        response = ChatMessage(
            text=f"Эхо: {message.text}",
            user="server"
        )
        yield response

Практический пример: Стриминг идеально подходит для передачи больших файлов, датчиков в реальном времени, чатов, обработки логов.

Урок 6: Безопасность и аутентификация

TLS/SSL шифрование

Для защищенной передачи данных используйте TLS (как HTTPS).

# Сервер с TLS
with open('server.key', 'rb') as f:
    private_key = f.read()
with open('server.crt', 'rb') as f:
    certificate_chain = f.read()

credentials = grpc.ssl_server_credentials(
    [(private_key, certificate_chain)]
)
server.add_secure_port('[::]:50051', credentials)

# Клиент с TLS
with open('ca.crt', 'rb') as f:
    ca_cert = f.read()

credentials = grpc.ssl_channel_credentials(ca_cert)
channel = grpc.secure_channel('localhost:50051', credentials)

Аутентификация через токены

Используйте метаданные (заголовки) для передачи токенов.

metadata = [('authorization', 'Bearer my-secret-token')]

def GetUser(self, request, context):

Интерсепторы (Interceptors)

Интерсепторы позволяют перехватывать запросы для логирования, аутентификации, метрик.

# Интерсептор для логирования
class LoggingInterceptor(grpc.ServerInterceptor):
    def intercept_service(self, continuation, handler_call_details):
        print(f"Вызов метода: {handler_call_details.method}")
        return continuation(handler_call_details)

# Подключение интерсептора
server = grpc.server(
    futures.ThreadPoolExecutor(max_workers=10),
    interceptors=[LoggingInterceptor()]
)

Урок 8: Практический проект — Микросервис на gRPC

Проект: Заметки с gRPC

Создадим простой микросервис для управления заметками с поддержкой стриминга.

# notes_service.proto
syntax = "proto3";

service NotesService {
    rpc CreateNote (CreateNoteRequest) returns (Note);
    rpc GetNote (GetNoteRequest) returns (Note);
    rpc ListNotes (ListNotesRequest) returns (stream Note);
    rpc WatchNotes (Empty) returns (stream NoteUpdate);
}

message Note {
    int32 id = 1;
    string title = 2;
    string content = 3;
    int64 created_at = 4;
}

message CreateNoteRequest {
    string title = 1;
    string content = 2;
}

message GetNoteRequest {
    int32 id = 1;
}

message ListNotesRequest {
    int32 limit = 1;
    int32 offset = 2;
}

message NoteUpdate {
    string action = 1;  // CREATE, UPDATE, DELETE
    Note note = 2;
}

message Empty {}
# notes_server.py
import grpc
import notes_service_pb2
import notes_service_pb2_grpc
from concurrent import futures
import time
import threading

class NotesServiceServicer(notes_service_pb2_grpc.NotesServiceServicer):
    def __init__(self):
        self.notes = {}
        self.next_id = 1
        self.watchers = []
        self.lock = threading.Lock()
    
    def CreateNote(self, request, context):
        with self.lock:
            note = notes_service_pb2.Note(
                id=self.next_id,
                title=request.title,
                content=request.content,
                created_at=int(time.time())
            )
            self.notes[self.next_id] = note
            self.next_id += 1
            
            # Уведомляем watchers
            update = notes_service_pb2.NoteUpdate(
                action="CREATE",
                note=note
            )
            for watcher in self.watchers:
                watcher.put(update)
            
            return note
    
    def GetNote(self, request, context):
        note = self.notes.get(request.id)
        if note:
            return note
        context.set_code(grpc.StatusCode.NOT_FOUND)
        return notes_service_pb2.Note()
    
    def ListNotes(self, request, context):
        notes_list = list(self.notes.values())
        start = request.offset
        end = start + request.limit if request.limit > 0 else len(notes_list)
        
        for note in notes_list[start:end]:
            yield note
    
    def WatchNotes(self, request, context):
        queue = queue.Queue()
        with self.lock:
            self.watchers.append(queue)
        
        try:
            while context.is_active():
                try:
                    update = queue.get(timeout=1)
                    yield update
                except queue.Empty:
                    continue
        finally:
            with self.lock:
                self.watchers.remove(queue)

def serve():
    server = grpc.server(futures.ThreadPoolExecutor(max_workers=10))
    notes_service_pb2_grpc.add_NotesServiceServicer_to_server(
        NotesServiceServicer(), server
    )
    server.add_insecure_port('[::]:50051')
    server.start()
    print("Notes Service запущен на порту 50051")
    server.wait_for_termination()

if __name__ == '__main__':
    serve()
