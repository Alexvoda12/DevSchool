Урок 1: Введение в NoSQL

Что такое NoSQL?

NoSQL (Not Only SQL) — это подход к хранению данных, отличный от традиционных реляционных баз данных. NoSQL базы данных не используют таблицы, строки и столбцы как основную структуру.

Когда использовать NoSQL:

  • Большие объемы данных (Big Data)
  • Гибкая структура данных (схема не фиксирована)
  • Высокая масштабируемость и производительность
  • Работа с неструктурированными данными
  • Быстрая разработка (agile)
Характеристика SQL (реляционные) NoSQL
СтруктураТаблицы, строки, столбцыДокументы, ключ-значение, графы, колонки
СхемаФиксированная (строгая)Гибкая (schema-less)
МасштабированиеВертикальноеГоризонтальное
ТранзакцииACIDBASE (eventual consistency)
ПримерыPostgreSQL, MySQLMongoDB, Redis, Cassandra

Типы NoSQL баз данных

  • Документо-ориентированные (MongoDB, CouchDB) — хранят данные в формате JSON/BSON
  • Ключ-значение (Redis, DynamoDB) — простые и быстрые хранилища
  • Колоночные (Cassandra, HBase) — оптимизированы для аналитики
  • Графовые (Neo4j, ArangoDB) — для работы со связями между данными

Урок 2: MongoDB — документо-ориентированная БД

Установка и подключение

# Установка драйвера
pip install pymongo

# Подключение к MongoDB
from pymongo import MongoClient

client = MongoClient('mongodb://localhost:27017/')
db = client['mydatabase']
collection = db['users']

CRUD операции в MongoDB

# CREATE (вставка)
user = {"name": "Анна", "age": 25, "city": "Москва"}
result = collection.insert_one(user)
print(result.inserted_id)

# Множественная вставка
users = [
    {"name": "Иван", "age": 30},
    {"name": "Мария", "age": 28}
]
collection.insert_many(users)

# READ (чтение)
one = collection.find_one({"name": "Анна"})
all_users = collection.find()
for u in all_users:
    print(u)

# Фильтрация
adults = collection.find({"age": {"$gte": 18}})

# UPDATE
collection.update_one(
    {"name": "Анна"},
    {"$set": {"age": 26, "city": "СПб"}}
)

# DELETE
collection.delete_one({"name": "Анна"})
collection.delete_many({"age": {"$lt": 18}})

Сложные запросы и агрегации

# Агрегация — группировка и подсчет
pipeline = [
    {"$match": {"age": {"$gte": 18}}},
    {"$group": {"_id": "$city", "count": {"$sum": 1}, "avg_age": {"$avg": "$age"}}},
    {"$sort": {"count": -1}}
]
results = collection.aggregate(pipeline)

# Индексы для ускорения запросов
collection.create_index("name")
collection.create_index([("age", -1)])  # сортировка по убыванию

Совет: Используйте find().limit(10) для ограничения количества документов, особенно в production.

Урок 3: Redis — хранилище ключ-значение

Быстрое кэширование и сессии

# Установка
pip install redis

import redis

# Подключение
r = redis.Redis(host='localhost', port=6379, decode_responses=True)

# Базовые операции
r.set('user:100', '{"name": "Анна", "age": 25}')
user_data = r.get('user:100')
print(user_data)

# Установка с таймаутом (TTL)
r.setex('session:abc123', 3600, 'user_id=100')

# Инкремент (счетчики)
r.incr('page_views')
r.incrby('page_views', 5)

# Работа с хэшами (структуры)
r.hset('user:101', mapping={'name': 'Иван', 'age': '30'})
r.hset('user:101', 'city', 'Москва')
user = r.hgetall('user:101')

# Списки (очереди)
r.lpush('notifications', 'message1', 'message2')
r.rpush('notifications', 'message3')
latest = r.lpop('notifications')

# Множества (уникальные элементы)
r.sadd('tags:python', 'web', 'data', 'ml')
r.sadd('tags:python', 'web')  # дубль не добавится
tags = r.smembers('tags:python')

Практические сценарии

# 1. Кэширование результатов API
def get_user(user_id):
    cache_key = f"user:{user_id}"
    cached = r.get(cache_key)
    if cached:
        return eval(cached)
    # Симуляция запроса к БД
    user = {"id": user_id, "name": f"User {user_id}"}
    r.setex(cache_key, 300, str(user))  # кэш на 5 минут
    return user

# 2. Rate limiting (ограничение запросов)
def rate_limit(user_id, limit=10, period=60):
    key = f"rate:{user_id}"
    current = r.incr(key)
    if current == 1:
        r.expire(key, period)
    return current <= limit

# 3. Сессии пользователей
def create_session(user_id):
    session_id = str(uuid.uuid4())
    r.hset(f"session:{session_id}", mapping={"user_id": user_id, "created": str(datetime.now())})
    r.expire(f"session:{session_id}", 86400)  # 1 день
    return session_id

Урок 4: Cassandra — колоночная БД

Особенности и работа с Cassandra

# Установка драйвера
pip install cassandra-driver

from cassandra.cluster import Cluster

cluster = Cluster(['127.0.0.1'])
session = cluster.connect()

# Создание keyspace и таблицы
session.execute("""
    CREATE KEYSPACE IF NOT EXISTS myapp
    WITH replication = {'class': 'SimpleStrategy', 'replication_factor': 1}
""")
session.set_keyspace('myapp')

session.execute("""
    CREATE TABLE IF NOT EXISTS users (
        user_id UUID PRIMARY KEY,
        name text,
        email text,
        created_at timestamp
    )
""")

# Вставка данных
from uuid import uuid4
session.execute("""
    INSERT INTO users (user_id, name, email, created_at)
    VALUES (%s, %s, %s, %s)
""", (uuid4(), "Анна", "anna@example.com", datetime.now()))

# Чтение данных
rows = session.execute("SELECT * FROM users")
for row in rows:
    print(row.user_id, row.name, row.email)

Важно: Cassandra оптимизирована для записи и чтения по ключу. Избегайте запросов без WHERE с partition key.
