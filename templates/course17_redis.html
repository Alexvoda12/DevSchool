Урок 1: Что такое Redis и зачем он нужен?

Что такое Redis?

Redis (REmote DIctionary Server) — это высокопроизводительное хранилище данных типа "ключ-значение" в оперативной памяти. Оно работает как база данных, кэш и брокер сообщений.

Ключевые особенности:

  • Скорость: данные хранятся в RAM, поэтому операции выполняются за микросекунды
  • Разнообразные структуры данных: строки, хэши, списки, множества, отсортированные множества
  • Постоянство: данные можно сохранять на диск
  • Репликация: поддержка master-slave репликации
  • Транзакции: атомарные операции
  • Pub/Sub: система сообщений издатель-подписчик

Когда использовать Redis?

Сценарий использования Пример
Кэширование Кэширование результатов запросов к БД, HTML страниц
Хранение сессий Сохранение данных пользовательских сессий в веб-приложениях
Очереди задач Брокер для фоновых задач (Celery, RQ)
Рейтинги и лидерборды Хранение отсортированных списков (Sorted Sets)
Лимиты и счетчики Rate limiting, счетчики просмотров
Реальном времени данные Чаты, уведомления, ленты новостей

Redis vs традиционные базы данных

Характеристика Redis PostgreSQL/MySQL
Хранение В оперативной памяти На диске
Скорость чтения Микросекунды (~0.1 мс) Миллисекунды (~1-10 мс)
Структуры данных Богатые (строки, списки, множества) Таблицы, JSON (PostgreSQL)
Сложные запросы Ограничены Полноценный SQL
Постоянство Опциональное Гарантированное
Объем данных Ограничен RAM Ограничен диском (TB+)

Совет: Redis обычно используется как дополнительное хранилище для ускорения, а не замена основной БД. Комбинируйте Redis для "горячих" данных и PostgreSQL/MySQL для постоянного хранения.

Урок 3: Структуры данных: строки и хэши

Строки (Strings)

Строки — самая простая структура в Redis. Могут хранить текст, числа, JSON, сериализованные объекты.

import redis
import json

r = redis.Redis(decode_responses=True)

# Сохранение JSON
user_data = {
    'id': 1,
    'name': 'Анна',
    'age': 25,
    'email': 'anna@example.com'
}
r.set('user:1', json.dumps(user_data))

# Получение и десериализация
data = r.get('user:1')
if data:
    user = json.loads(data)
    print(user['name'])  # Анна

# Сохранение бинарных данных (изображения, файлы)
with open('avatar.jpg', 'rb') as f:
    r.set('avatar:1', f.read())

# Получение части строки (подстрока)
r.set('long_text', 'Hello, World!')
print(r.getrange('long_text', 0, 4))  # Hello

# Добавление в конец строки
r.append('long_text', ' Welcome!')
print(r.get('long_text'))  # Hello, World! Welcome!

Хэши (Hashes)

Хэши — это словари, где можно хранить множество полей под одним ключом. Идеально для объектов.

import redis
r = redis.Redis(decode_responses=True)

# Сохранение хэша
r.hset('user:100', mapping={
    'name': 'Иван',
    'email': 'ivan@example.com',
    'age': 30,
    'city': 'Москва'
})

# Получение всех полей
user = r.hgetall('user:100')
print(user)  # {'name': 'Иван', 'email': 'ivan@example.com', ...}

# Получение одного поля
name = r.hget('user:100', 'name')
print(name)  # Иван

# Получение нескольких полей
fields = r.hmget('user:100', ['name', 'email'])
print(fields)  # ['Иван', 'ivan@example.com']

# Получение всех ключей/значений
keys = r.hkeys('user:100')
values = r.hvals('user:100')
print(keys)    # ['name', 'email', 'age', 'city']
print(values)  # ['Иван', 'ivan@example.com', '30', 'Москва']

# Увеличение числового поля
r.hincrby('user:100', 'age', 1)  # Увеличить возраст на 1

# Проверка существования поля
exists = r.hexists('user:100', 'name')
print(exists)  # True

# Удаление полей
r.hdel('user:100', 'city')

# Длина хэша (количество полей)
print(r.hlen('user:100'))  # 3

Сравнение: Строка vs Хэш

Сценарий Строка (JSON) Хэш
Хранит объект Сериализованный JSON Отдельные поля
Получить все поля 1 операция (получить JSON) 1 операция (hgetall)
Получить одно поле Нужно десериализовать весь JSON 1 операция (hget)
Обновить одно поле Нужно обновить весь JSON 1 операция (hset)
Удалить поле Нужно обновить весь JSON 1 операция (hdel)
Использование памяти Меньше служебных данных Немного больше

Совет: Используйте хэши для объектов, к которым часто обращаются по отдельным полям. Используйте строки с JSON, если нужно атомарно получить весь объект и он редко изменяется.

Списки (Lists)

Списки Redis — это связанные списки. Идеально для очередей и стеков.

import redis
r = redis.Redis(decode_responses=True)

# Добавление в начало/конец списка
r.lpush('messages', 'сообщение 1')   # В начало
r.rpush('messages', 'сообщение 2')   # В конец
r.rpush('messages', 'сообщение 3')

# Получение всех элементов
messages = r.lrange('messages', 0, -1)
print(messages)  # ['сообщение 1', 'сообщение 2', 'сообщение 3']

# Получение и удаление из начала/конца (для очередей)
msg = r.lpop('messages')      # Извлечь из начала
print(msg)  # сообщение 1
msg = r.rpop('messages')       # Извлечь из конца
print(msg)  # сообщение 3

# Длина списка
length = r.llen('messages')
print(length)  # 1

# Получение элемента по индексу
second = r.lindex('messages', 0)

# Обрезать список (оставить только первые N)

Множества (Sets)

Множества — неупорядоченные коллекции уникальных элементов. Идеально для тегов, уникальных идентификаторов.

import redis
r = redis.Redis(decode_responses=True)

# Добавление элементов
r.sadd('tags', 'python', 'redis', 'programming', 'python')  # 'python' добавится один раз

# Получение всех элементов
tags = r.smembers('tags')
print(tags)  # {'python', 'redis', 'programming'}

# Проверка принадлежности
has_python = r.sismember('tags', 'python')
print(has_python)  # True

# Удаление элемента
r.srem('tags', 'programming')

# Количество элементов
count = r.scard('tags')
print(count)  # 2

# Случайный элемент
random_tag = r.srandmember('tags')
print(random_tag)  # python или redis

# Операции над множествами
r.sadd('set1', 'a', 'b', 'c')
r.sadd('set2', 'b', 'c', 'd')

# Пересечение
intersection = r.sinter('set1', 'set2')
print(intersection)  # {'b', 'c'}

# Объединение
union = r.sunion('set1', 'set2')
print(union)  # {'a', 'b', 'c', 'd'}

# Разность
diff = r.sdiff('set1', 'set2')
print(diff)  # {'a'}

Отсортированные множества (Sorted Sets)

Каждый элемент имеет вес (score), по которому элементы сортируются. Идеально для рейтингов, лидербордов, очередей с приоритетом.

import redis
r = redis.Redis(decode_responses=True)

# Добавление с весом
r.zadd('leaderboard', {
    'Иван': 1500,
    'Анна': 2300,
    'Петр': 800,
    'Мария': 1950,
    'Сергей': 3100
})

# Получение всех в порядке возрастания
all_players = r.zrange('leaderboard', 0, -1, withscores=True)
print(all_players)  # [('Петр', 800.0), ('Иван', 1500.0), ('Мария', 1950.0), ...]

# Получение в порядке убывания (топ)
top_players = r.zrevrange('leaderboard', 0, 2, withscores=True)
print(top_players)  # [('Сергей', 3100.0), ('Анна', 2300.0), ('Мария', 1950.0)]

# Получение ранга (места)
rank = r.zrevrank('leaderboard', 'Анна')
print(f"Анна на {rank + 1} месте")  # Анна на 2 месте

# Получение очков игрока
score = r.zscore('leaderboard', 'Сергей')
print(f"Сергей: {score} очков")  # Сергей: 3100 очков

# Увеличение очков
r.zincrby('leaderboard', 100, 'Иван')

# Получение игроков в диапазоне очков
players_in_range = r.zrangebyscore('leaderboard', 1500, 2500, withscores=True)
print(players_in_range)

# Количество игроков в диапазоне
count = r.zcount('leaderboard', 1000, 3000)
print(count)

# Удаление игрока
r.zrem('leaderboard', 'Петр')

    key = f'search_history:{user_id}'

def rate_limit(user_id, max_requests, window_seconds):
    key = f'rate_limit:{user_id}'
    now = time.time()
    # Удаляем старые записи

    # Добавляем текущий запрос
    r.zadd(key, {str(now): now})
    # Проверяем количество запросов
    count = r.zcard(key)
    return count <= max_requests

Стратегии кэширования

1. Cache-Aside (Lazy Loading)

def get_user(user_id):
    # Сначала проверяем кэш
    cache_key = f'user:{user_id}'
    cached_user = r.get(cache_key)
    
    if cached_user:
        print("Данные из кэша")
        return json.loads(cached_user)
    
    # Если нет в кэше — запрашиваем из БД
    print("Запрос к БД")
    user = db.query(User).filter_by(id=user_id).first()
    
    if user:
        # Сохраняем в кэш на 5 минут
        r.setex(cache_key, 300, json.dumps(user.to_dict()))
    
    return user

2. Write-Through

def update_user(user_id, data):
    # Обновляем в БД
    db_user = db.query(User).filter_by(id=user_id).first()
    db_user.update(data)
    db.commit()
    
    # Сразу обновляем кэш
    cache_key = f'user:{user_id}'
    r.setex(cache_key, 300, json.dumps(db_user.to_dict()))

3. Write-Behind

def update_user_async(user_id, data):
    # Сохраняем только в кэш
    cache_key = f'user:{user_id}'
    r.hset(cache_key, mapping=data)
    
    # Добавляем задачу в очередь для фоновой записи в БД
    r.lpush('db_update_queue', json.dumps({
        'user_id': user_id,
        'data': data
    }))

Паттерны кэширования

import hashlib
import json

class RedisCache:
    def __init__(self, redis_client, default_ttl=300):
        self.r = redis_client
        self.default_ttl = default_ttl
    
    def get_or_set(self, key, callback, ttl=None):
        """Получить из кэша или выполнить callback и сохранить"""
        cached = self.r.get(key)
        if cached:
            return json.loads(cached)
        
        result = callback()
        if result:
            self.r.setex(key, ttl or self.default_ttl, json.dumps(result))
        return result
    
    def cache_function(self, ttl=None):
        """Декоратор для кэширования результатов функции"""
        def decorator(func):
            def wrapper(*args, **kwargs):
                # Создаем ключ на основе имени функции и аргументов
                key_data = f"{func.__name__}:{args}:{kwargs}"
                key = hashlib.md5(key_data.encode()).hexdigest()
                key = f"func_cache:{key}"
                
                return self.get_or_set(key, lambda: func(*args, **kwargs), ttl)
            return wrapper
        return decorator

# Использование
cache = RedisCache(r)

@cache.cache_function(ttl=60)  # Кэшировать на 60 секунд
def get_expensive_data(user_id):
    # Долгие вычисления или запрос к API
    import time
    time.sleep(2)
    return {"user_id": user_id, "data": "some_data"}

# Первый вызов — 2 секунды
result1 = get_expensive_data(1)
# Второй вызов — мгновенно
result2 = get_expensive_data(1)

Инвалидация кэша

class CacheInvalidation:
    def __init__(self, redis_client):
        self.r = redis_client
    
    def invalidate_user_cache(self, user_id):
        """Очистка кэша пользователя"""
        # Удаляем конкретный ключ
        self.r.delete(f'user:{user_id}')
        
        # Удаляем все ключи по паттерну
        pattern = f'user_posts:{user_id}:*'
        keys = self.r.keys(pattern)
        if keys:
            self.r.delete(*keys)
    
    def invalidate_pattern(self, pattern):
        """Очистка всех ключей по паттерну"""
        keys = self.r.keys(pattern)
        if keys:
            self.r.delete(*keys)
    
    def versioned_cache(self, prefix, version=1):
        """Версионирование кэша для массовой инвалидации"""
        def get_key(key):
            return f"{prefix}:v{version}:{key}"
        
        return get_key

# Использование
cache = CacheInvalidation(r)

# При обновлении данных
def update_user_profile(user_id, new_data):
    # Обновляем БД
    db.update_user(user_id, new_data)
    # Инвалидируем кэш
    cache.invalidate_user_cache(user_id)

Проблемы кэширования и их решения:

  • Cache Stampede: Много запросов одновременно к истекающему ключу → Используйте блокировки или "заглушки"
  • Cache Penetration: Запросы к несуществующим данным → Кэшируйте "пустые" результаты
  • Cache Avalanche: Массовое истечение ключей → Добавляйте случайное смещение к TTL

import uuid

import redis

class SessionManager:
    def __init__(self, redis_client, session_ttl=3600):
        self.r = redis_client
        self.session_ttl = session_ttl  # время жизни сессии в секундах
    
    def create_session(self, user_id, user_data=None):
        """Создание новой сессии"""
        session_id = str(uuid.uuid4())
        session_key = f'session:{session_id}'
        
        session_data = {
            'user_id': user_id,
            'created_at': int(time.time()),
            'user_agent': user_data.get('user_agent') if user_data else None,
            'ip': user_data.get('ip') if user_data else None,
            'data': user_data or {}
        }
        
        self.r.hset(session_key, mapping={
            'user_id': user_id,
            'data': json.dumps(session_data['data'])
        })
        self.r.expire(session_key, self.session_ttl)
        
        # Сохраняем связь user -> sessions
        self.r.sadd(f'user_sessions:{user_id}', session_id)
        
        return session_id
    
    def get_session(self, session_id):
        """Получение данных сессии"""
        session_key = f'session:{session_id}'
        
        if not self.r.exists(session_key):
            return None
        
        user_id = self.r.hget(session_key, 'user_id')
        data = self.r.hget(session_key, 'data')
        
        # Обновляем TTL при активности
        self.r.expire(session_key, self.session_ttl)
        
        return {
            'session_id': session_id,
            'user_id': user_id,
            'data': json.loads(data) if data else {}
        }
    
    def update_session_data(self, session_id, new_data):
        """Обновление данных сессии"""
        session_key = f'session:{session_id}'
        
        if not self.r.exists(session_key):
            return False
        
        current_data = self.r.hget(session_key, 'data')
        if current_data:
            data = json.loads(current_data)
            data.update(new_data)
            self.r.hset(session_key, 'data', json.dumps(data))
        else:
            self.r.hset(session_key, 'data', json.dumps(new_data))
        
        self.r.expire(session_key, self.session_ttl)
        return True
    
    def destroy_session(self, session_id):
        """Удаление сессии"""
        session_key = f'session:{session_id}'
        user_id = self.r.hget(session_key, 'user_id')
        
        if user_id:
            self.r.srem(f'user_sessions:{user_id}', session_id)
        
        self.r.delete(session_key)
    
    def destroy_all_user_sessions(self, user_id, except_session=None):
        """Удаление всех сессий пользователя"""
        sessions_key = f'user_sessions:{user_id}'
        sessions = self.r.smembers(sessions_key)
        
        for session_id in sessions:
            if session_id != except_session:
                self.destroy_session(session_id)
        
        self.r.delete(sessions_key)
    
    def get_active_sessions_count(self, user_id):
        """Количество активных сессий пользователя"""
        return self.r.scard(f'user_sessions:{user_id}')

import json
import threading
import time
import redis

class TaskQueue:
    def __init__(self, redis_client, queue_name='tasks'):
        self.r = redis_client
        self.queue_name = queue_name
        self.workers = []
        self.running = False
    
    def add_task(self, task_type, **kwargs):
        """Добавить задачу в очередь"""
        task = {
            'type': task_type,
            'params': kwargs,
            'created_at': time.time()
        }
        self.r.rpush(self.queue_name, json.dumps(task))
        print(f"Задача добавлена: {task_type}")
    
    def worker(self, worker_id):
        """Воркер, выполняющий задачи"""
        print(f"Воркер {worker_id} запущен")
        
        while self.running:
            # Блокирующая операция с таймаутом
            task_data = self.r.blpop(self.queue_name, timeout=1)
            
            if task_data:
                _, task_json = task_data
                task = json.loads(task_json)
                
                print(f"Воркер {worker_id} обрабатывает: {task['type']}")
                
                # Выполняем задачу
                try:
                    self.execute_task(task)
                    print(f"Воркер {worker_id} завершил: {task['type']}")
                except Exception as e:
                    print(f"Ошибка в задаче {task['type']}: {e}")
                    # Можно добавить в очередь ошибок
                    self.r.rpush(f'{self.queue_name}:errors', task_json)
    
    def execute_task(self, task):
        """Выполнение задачи в зависимости от типа"""
        task_type = task['type']
        params = task['params']
        
        if task_type == 'send_email':
            self.send_email(**params)
        elif task_type == 'process_image':
            self.process_image(**params)
        elif task_type == 'generate_report':
            self.generate_report(**params)
        else:
            raise ValueError(f"Неизвестный тип задачи: {task_type}")
    
    def send_email(self, to, subject, body):
        """Отправка email (заглушка)"""
        time.sleep(1)  # Имитация отправки
        print(f"  📧 Email отправлен на {to}")
    
    def process_image(self, image_path, size):
        """Обработка изображения (заглушка)"""
        time.sleep(2)  # Имитация обработки
        print(f"  🖼️ Изображение {image_path} обработано до {size}")
    
    def generate_report(self, report_type, date):
        """Генерация отчета (заглушка)"""
        time.sleep(3)  # Имитация генерации
        print(f"  📊 Отчет {report_type} за {date} сгенерирован")
    
    def start_workers(self, count=3):
        """Запуск воркеров"""
        self.running = True
        
        for i in range(count):
            worker = threading.Thread(target=self.worker, args=(i,))
            worker.daemon = True
            worker.start()
            self.workers.append(worker)
    
    def stop_workers(self):
        """Остановка воркеров"""
        self.running = False

# Использование
queue = TaskQueue(r)
queue.start_workers(2)

# Добавляем задачи

class PriorityTaskQueue:
    def __init__(self, redis_client):
        self.r = redis_client
    
    def add_task(self, priority, task_type, **kwargs):
        """
        Добавить задачу с приоритетом
        priority: 0 (высокий) - 10 (низкий)
        """
        task = {
            'type': task_type,
            'params': kwargs,
            'created_at': time.time()
        }
        score = priority  # Чем меньше score, тем выше приоритет
        self.r.zadd('priority_tasks', {json.dumps(task): score})
    
    def get_next_task(self):
        """Получить следующую задачу (с наивысшим приоритетом)"""
        # Получаем задачу с наименьшим score
        tasks = self.r.zrange('priority_tasks', 0, 0, withscores=True)
        
        if tasks:
            task_json, score = tasks[0]
            # Удаляем из очереди
            self.r.zrem('priority_tasks', task_json)
            return json.loads(task_json)
        
        return None
    
    def worker(self):
        """Воркер с обработкой по приоритету"""
        while True:
            task = self.get_next_task()
            if task:
                self.execute_task(task)
            else:
                time.sleep(0.1)  # Нет задач

# Использование
