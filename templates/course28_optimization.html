Урок 1: Введение в профилирование

Почему код может работать медленно?

Профилирование — это процесс измерения производительности кода. Оно помогает ответить на вопросы: "Какая функция выполняется дольше всего?", "Где тратится больше всего памяти?", "Почему программа тормозит?".

Без профилирования мы оптимизируем наугад, часто тратя время на неважные участки кода.

Виды профилирования

  • Профилирование времени (CPU): измеряет, сколько времени занимает выполнение каждой функции.
  • Профилирование памяти (Memory): отслеживает, сколько памяти использует код.
  • Профилирование вызовов (Call Graph): показывает, как функции вызывают друг друга.

В этом курсе мы изучим инструменты для каждого типа профилирования.

Золотое правило оптимизации: "Сначала измерь, потом оптимизируй". Никогда не предполагай, что медленно. Всегда проверяй с помощью профилировщика.

Урок 2: cProfile — профилирование CPU

Использование cProfile

cProfile — встроенный модуль Python для профилирования. Он показывает, сколько времени тратится на каждую функцию.

import cProfile
import pstats
import io

def slow_function():
    total = 0
    for i in range(1000000):
        total += i ** 2
    return total

def fast_function():
    return sum(i ** 2 for i in range(1000000))

def main():
    slow_function()
    fast_function()

# Профилирование через командную строку
# python -m cProfile my_script.py

# Профилирование из кода
profiler = cProfile.Profile()
profiler.enable()
main()
profiler.disable()

# Сохранение и анализ результатов
stream = io.StringIO()
stats = pstats.Stats(profiler, stream=stream)
stats.sort_stats('cumulative')  # Сортировка по накопленному времени
stats.print_stats(10)  # Вывод топ-10 функций

print(stream.getvalue())

Интерпретация результатов

ПолеОписание
ncallsКоличество вызовов функции
tottimeОбщее время, проведённое в функции (без вызовов подфункций)
percall (tottime)Среднее время одного вызова функции
cumtimeОбщее время, включая время во всех подфункциях
percall (cumtime)Среднее время одного вызова с учётом подфункций

Важно! Смотрите на tottime — это время, которое функция потратила на свою работу, не считая вызовы других функций. Именно здесь чаще всего находятся узкие места.

Визуализация с snakeviz

Для визуализации результатов установите snakeviz: pip install snakeviz.

# Генерация файла со статистикой
import cProfile
cProfile.run('main()', 'profile_stats.prof')

# Запуск визуализации
# snakeviz profile_stats.prof

Урок 3: Профилирование памяти

Библиотека memory_profiler

Установка: pip install memory-profiler

from memory_profiler import profile
import time

@profile
def memory_intensive():
    # Создаём большой список
    big_list = [i for i in range(1000000)]
    
    # Добавляем ещё один
    another_list = [x * 2 for x in big_list]
    
    # Освобождаем память
    del big_list
    
    # Создаём словарь
    big_dict = {i: i**2 for i in range(500000)}
    
    return len(another_list) + len(big_dict)

if __name__ == '__main__':
    result = memory_intensive()
    print(f"Результат: {result}")

Урок 4: Оптимизация алгоритмов

Сложность алгоритмов (Big O)

Выбор правильного алгоритма — самый эффективный способ ускорить код. Ошибка в сложности может замедлить программу в тысячи раз.

СложностьНазваниеПример
O(1)КонстантнаяДоступ по индексу
O(log n)ЛогарифмическаяБинарный поиск
O(n)ЛинейнаяПоиск элемента
O(n log n)Линейно-логарифмическаяБыстрая сортировка
O(n²)КвадратичнаяВложенные циклы
O(2ⁿ)ЭкспоненциальнаяРекурсивный перебор

Практические примеры оптимизации

# Плохо: O(n²)
def find_duplicates_slow(items):
    duplicates = []
    for i in range(len(items)):
        for j in range(i+1, len(items)):
            if items[i] == items[j] and items[i] not in duplicates:
                duplicates.append(items[i])
    return duplicates

# Хорошо: O(n) с использованием set
def find_duplicates_fast(items):
    seen = set()
    duplicates = set()
    for item in items:
        if item in seen:
            duplicates.add(item)
        else:
            seen.add(item)
    return list(duplicates)

# Плохо: частый поиск в списке
def slow_member_check(items, target):
    return target in items  # O(n) каждый раз

# Хорошо: использовать set для проверки
def fast_member_check(items, target):
    items_set = set(items)  # O(n) один раз
    return target in items_set  # O(1)

Оптимизация работы с данными

  • Используйте генераторы вместо списков для больших данных.
  • Избегайте многократных вычислений — кэшируйте результаты (functools.lru_cache).
  • Выбирайте правильные структуры данных: set для уникальности, deque для очередей, heap для приоритетов.

Урок 5: Работа с GIL

Что такое GIL?

GIL (Global Interpreter Lock) — глобальная блокировка интерпретатора, которая позволяет выполнять только один поток Python одновременно. Это ограничение для CPU-bound задач.

GIL не мешает I/O-bound задачам (сеть, файлы), но CPU-bound задачи в потоках не ускоряются.

Обход GIL

import threading
import multiprocessing
import time

# CPU-bound задача
def cpu_intensive(n):
    return sum(i * i for i in range(n))

# Потоки НЕ ускоряют CPU-bound задачи из-за GIL
def test_threads():
    numbers = [10_000_000] * 4
    start = time.time()
    with threading.ThreadPoolExecutor(max_workers=4) as executor:
        results = list(executor.map(cpu_intensive, numbers))
    print(f"Threads: {time.time() - start:.2f}s")

# Процессы ускоряют за счёт отдельных интерпретаторов
def test_processes():
    numbers = [10_000_000] * 4
    start = time.time()
    with multiprocessing.Pool(processes=4) as pool:
        results = pool.map(cpu_intensive, numbers)
    print(f"Processes: {time.time() - start:.2f}s")

Вывод: Для CPU-bound задач используйте multiprocessing или выносите тяжелые вычисления в C-расширения (NumPy, Cython). Для I/O-bound задач асинхронность и потоки работают отлично.
