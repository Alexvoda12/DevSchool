Урок 1: Введение в FastAPI

Что такое FastAPI?

FastAPI — это современный, быстрый (высокопроизводительный) веб-фреймворк для создания API на Python 3.8+ на основе стандартных подсказок типов Python. Он создан для того, чтобы разработка API была простой, быстрой и надежной.

Основные преимущества FastAPI:

  • Высокая производительность: Один из самых быстрых фреймворков Python, благодаря Starlette и Pydantic
  • Автоматическая документация: Генерирует документацию OpenAPI (Swagger UI и ReDoc) автоматически
  • Асинхронность из коробки: Поддержка async/await без дополнительных настроек
  • Валидация данных: Использует Pydantic для автоматической валидации
  • Внедрение зависимостей: Мощная и простая система DI

Установка FastAPI и сервера Uvicorn:

# Установка FastAPI
pip install fastapi

# Установка сервера ASGI (Uvicorn)
pip install uvicorn

Первое приложение

Создадим простое FastAPI приложение с одним эндпоинтом.

# main.py
from fastapi import FastAPI

# Создаем экземпляр приложения
app = FastAPI()

# Определяем GET эндпоинт на корневом пути "/"
@app.get("/")
async def root():
    return {"message": "Hello, World!"}

# Определяем эндпоинт с параметром пути
@app.get("/items/{item_id}")
async def read_item(item_id: int):
    return {"item_id": item_id}

Запуск приложения:

# Запуск сервера
uvicorn main:app --reload

После запуска вы можете открыть в браузере:

  • http://localhost:8000 — ваш API
  • http://localhost:8000/docs — автоматическая документация Swagger UI
  • http://localhost:8000/redoc — альтернативная документация ReDoc

Почему Uvicorn? FastAPI — это ASGI-фреймворк (Asynchronous Server Gateway Interface), в отличие от классического WSGI (как у Flask). Uvicorn — это ASGI-сервер, который позволяет использовать все преимущества асинхронности.

Урок 2: Pydantic модели и валидация

Модели данных Pydantic

Pydantic — это библиотека для валидации данных на основе аннотаций типов. FastAPI использует Pydantic для определения структуры запросов и ответов.

from fastapi import FastAPI
from pydantic import BaseModel
from typing import Optional

app = FastAPI()

# Определяем Pydantic модель для данных пользователя
class User(BaseModel):
    id: int
    name: str
    email: str
    age: Optional[int] = None  # Поле необязательное
    is_active: bool = True      # Поле со значением по умолчанию

# Модель для создания пользователя (без id)
class UserCreate(BaseModel):
    name: str
    email: str
    age: Optional[int] = None

@app.post("/users/")
async def create_user(user: UserCreate):
    # Автоматическая валидация данных происходит здесь
    # Если данные не соответствуют модели, вернется 422 Unprocessable Entity
    return {"message": f"User {user.name} created", "data": user}

Валидация с Field

Для более сложной валидации можно использовать Field из Pydantic.

from pydantic import BaseModel, Field, EmailStr, validator
import re

class Product(BaseModel):
    name: str = Field(..., min_length=3, max_length=50, description="Название товара")
    price: float = Field(..., gt=0, le=10000, description="Цена товара")
    quantity: int = Field(..., ge=0, le=1000, description="Количество на складе")
    email: EmailStr  # Специальный тип для валидации email
    
    @validator('name')
    def name_must_contain_letters(cls, v):
        if not re.match(r'^[a-zA-Zа-яА-Я0-9\s]+$', v):
            raise ValueError('Имя может содержать только буквы, цифры и пробелы')
        return v

@app.post("/products/")
async def create_product(product: Product):
    return product

Урок 3: Эндпоинты и параметры

HTTP методы

FastAPI поддерживает все стандартные HTTP методы:

from fastapi import FastAPI, Query, Path, Body

app = FastAPI()

# GET — получение данных
@app.get("/items/")
async def read_items(skip: int = 0, limit: int = 10):
    return {"skip": skip, "limit": limit}

# POST — создание данных
@app.post("/items/")
async def create_item(item: dict):
    return {"item": item}

# PUT — обновление (полная замена)
@app.put("/items/{item_id}")
async def update_item(item_id: int, item: dict):
    return {"item_id": item_id, "item": item}

# PATCH — частичное обновление
@app.patch("/items/{item_id}")
async def patch_item(item_id: int, item: dict):
    return {"item_id": item_id, "item": item}

# DELETE — удаление
@app.delete("/items/{item_id}")
async def delete_item(item_id: int):
    return {"deleted": item_id}

Типы параметров

FastAPI автоматически определяет, откуда брать параметры:

  • Параметры пути (Path) — из URL: /items/{item_id}
  • Параметры запроса (Query) — из строки запроса: ?skip=0&limit=10
  • Тело запроса (Body) — из JSON тела POST, PUT, PATCH запросов
  • Заголовки (Header) — из HTTP заголовков
  • Cookie — из кук
from fastapi import FastAPI, Query, Path, Header, Cookie, Body
from typing import Optional

app = FastAPI()

@app.get("/items/{item_id}")
async def read_item(
    # Параметр пути
    item_id: int = Path(..., title="ID предмета", ge=1),
    
    # Параметры запроса
    q: Optional[str] = Query(None, max_length=50, title="Поисковый запрос"),
    page: int = Query(1, ge=1),
    size: int = Query(10, ge=1, le=100),
    
    # Заголовки
    user_agent: Optional[str] = Header(None),
    
    # Cookie
    session_id: Optional[str] = Cookie(None),
):
    return {
        "item_id": item_id,
        "q": q,
        "page": page,
        "size": size,
        "user_agent": user_agent,
        "session_id": session_id
    }

# Тело запроса
@app.post("/users/")
async def create_user(
    name: str = Body(..., embed=True),  # embed=True для одиночного поля
    email: str = Body(...)
):
    return {"name": name, "email": email}

Урок 4: Dependency Injection (DI)

Что такое зависимости?

Зависимости (Dependencies) — это способ повторно использовать код и разделять логику. Это могут быть функции, классы или даже другие эндпоинты.

from fastapi import FastAPI, Depends, HTTPException, status

app = FastAPI()

# Простая зависимость
async def common_parameters(q: str = None, skip: int = 0, limit: int = 100):
    return {"q": q, "skip": skip, "limit": limit}

# Зависимость для проверки аутентификации
async def get_current_user(token: str = Header(...)):
    if token != "secret-token":
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Invalid authentication credentials"
        )
    return {"username": "john_doe", "token": token}

# Использование зависимостей
@app.get("/items/")
async def read_items(params: dict = Depends(common_parameters)):
    return params

@app.get("/users/me")
async def read_users_me(current_user: dict = Depends(get_current_user)):
    return current_user

Зависимости с кэшированием

По умолчанию зависимости вычисляются один раз на запрос. Для зависимостей с кэшированием используйте параметр use_cache.

from fastapi import FastAPI, Depends, Request

app = FastAPI()

# Зависимость, которая может быть дорогой
async def get_db_session():
    # Имитация подключения к БД
    db = {"session_id": "12345"}
    yield db
    # Закрываем соединение после использования
    print("Closing DB connection")

# Зависимость для получения пользователя
async def get_current_user(request: Request, db=Depends(get_db_session)):
    # Получаем пользователя из БД
    user = {"id": 1, "name": "John"}
    request.state.user = user
    return user

@app.get("/profile/")
async def get_profile(user=Depends(get_current_user)):
    return user

@app.get("/dashboard/")
async def get_dashboard(user=Depends(get_current_user)):
    return {"user": user, "data": "Dashboard data"}

Урок 5: Обработка ошибок и middleware

Обработка ошибок

from fastapi import FastAPI, HTTPException, status, Request
from fastapi.responses import JSONResponse
from fastapi.exceptions import RequestValidationError

app = FastAPI()

# Кастомное исключение
class UserNotFoundError(Exception):
    def __init__(self, user_id: int):
        self.user_id = user_id
        self.message = f"User with id {user_id} not found"

# Обработчик кастомного исключения
@app.exception_handler(UserNotFoundError)
async def user_not_found_handler(request: Request, exc: UserNotFoundError):
    return JSONResponse(
        status_code=status.HTTP_404_NOT_FOUND,
        content={"detail": exc.message, "user_id": exc.user_id}
    )

# Обработчик стандартных HTTP исключений
@app.exception_handler(HTTPException)
async def http_exception_handler(request: Request, exc: HTTPException):
    return JSONResponse(
        status_code=exc.status_code,
        content={"detail": exc.detail}
    )

# Обработчик ошибок валидации
@app.exception_handler(RequestValidationError)
async def validation_exception_handler(request: Request, exc: RequestValidationError):
    return JSONResponse(
        status_code=status.HTTP_422_UNPROCESSABLE_ENTITY,
        content={"detail": exc.errors(), "body": exc.body}
    )

@app.get("/users/{user_id}")
async def get_user(user_id: int):
    if user_id != 1:
        raise UserNotFoundError(user_id)
    return {"id": 1, "name": "John"}

@app.get("/items/{item_id}")
async def get_item(item_id: int):
    if item_id > 100:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail="Item not found"
        )
    return {"id": item_id}

Middleware

Middleware — это компоненты, которые выполняются до и после обработки запроса.

import time
from fastapi import FastAPI, Request
from fastapi.middleware.cors import CORSMiddleware
from fastapi.middleware.trustedhost import TrustedHostMiddleware
from starlette.middleware.base import BaseHTTPMiddleware

app = FastAPI()

# CORS middleware (разрешает запросы с других доменов)
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # В продакшене лучше ограничить
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# Trusted Host middleware (защита от Host Header Injection)
app.add_middleware(
    TrustedHostMiddleware,
    allowed_hosts=["localhost", "127.0.0.1", "*.example.com"]
)

# Кастомный middleware для логирования
class LoggingMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        start_time = time.time()
        
        # Логируем запрос
        print(f"Request: {request.method} {request.url.path}")
        
        response = await call_next(request)
        
        # Логируем ответ
        process_time = time.time() - start_time
        print(f"Response: {response.status_code} - {process_time:.4f}s")
        
        response.headers["X-Process-Time"] = str(process_time)
        return response

app.add_middleware(LoggingMiddleware)

@app.get("/")
async def root():
    return {"message": "Hello"}

Урок 6: WebSockets и фоновые задачи

WebSockets

FastAPI поддерживает WebSockets для двусторонней связи с клиентом.

from fastapi import FastAPI, WebSocket, WebSocketDisconnect
from typing import List

app = FastAPI()

class ConnectionManager:
    def __init__(self):
        self.active_connections: List[WebSocket] = []
    
    async def connect(self, websocket: WebSocket):
        await websocket.accept()
        self.active_connections.append(websocket)
    
    def disconnect(self, websocket: WebSocket):
        self.active_connections.remove(websocket)
    
    async def broadcast(self, message: str):
        for connection in self.active_connections:
            await connection.send_text(message)

manager = ConnectionManager()

@app.websocket("/ws/{client_id}")
async def websocket_endpoint(websocket: WebSocket, client_id: str):
    await manager.connect(websocket)
    try:
        while True:
            data = await websocket.receive_text()
            await manager.broadcast(f"Client {client_id}: {data}")
    except WebSocketDisconnect:
        manager.disconnect(websocket)
        await manager.broadcast(f"Client {client_id} disconnected")

Фоновые задачи (Background Tasks)

Фоновые задачи выполняются после отправки ответа клиенту.

from fastapi import FastAPI, BackgroundTasks
import time

app = FastAPI()

def send_email(email: str, subject: str, body: str):
    """Имитация отправки email"""
    time.sleep(2)  # Имитация долгой операции
    print(f"Email sent to {email}: {subject}")

def process_file(file_path: str):
    """Имитация обработки файла"""
    time.sleep(5)
    print(f"File {file_path} processed")

@app.post("/users/")
async def create_user(email: str, background_tasks: BackgroundTasks):
    # Основной ответ отправляется сразу
    background_tasks.add_task(send_email, email, "Welcome!", "Thank you for registering!")
    return {"message": "User created, email will be sent soon"}

@app.post("/upload/")
async def upload_file(file_path: str, background_tasks: BackgroundTasks):
    background_tasks.add_task(process_file, file_path)
    return {"message": "File uploaded, processing in background"}

Фоновые задачи через asyncio.create_task

import asyncio
from fastapi import FastAPI

app = FastAPI()

# Глобальная задача
background_task = None

async def periodic_task():
    while True:
        print("Periodic task running...")
        await asyncio.sleep(10)

@app.on_event("startup")
async def startup_event():
    global background_task
    background_task = asyncio.create_task(periodic_task())

@app.on_event("shutdown")
async def shutdown_event():
    if background_task:
        background_task.cancel()

Урок 8: Практический проект — TODO API

Создание полноценного API

Создадим REST API для управления задачами (TODO list) с использованием FastAPI, Pydantic и in-memory хранилища.

from fastapi import FastAPI, HTTPException, status, Depends
from pydantic import BaseModel, Field
from typing import List, Optional
from uuid import uuid4, UUID

app = FastAPI(title="TODO API", version="1.0.0")

# Модели данных
class TodoCreate(BaseModel):
    title: str = Field(..., min_length=1, max_length=100)
    description: Optional[str] = Field(None, max_length=500)
    completed: bool = False

class TodoUpdate(BaseModel):
    title: Optional[str] = Field(None, min_length=1, max_length=100)
    description: Optional[str] = Field(None, max_length=500)
    completed: Optional[bool] = None

class Todo(TodoCreate):
    id: UUID
    created_at: str

# In-memory хранилище
todos_db: dict[UUID, dict] = {}

# Вспомогательная функция для получения todo
async def get_todo_or_404(todo_id: UUID):
    todo = todos_db.get(todo_id)
    if not todo:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail=f"Todo with id {todo_id} not found"
        )
    return todo

# GET /todos - получить все задачи
@app.get("/todos", response_model=List[Todo])
async def get_all_todos(completed: Optional[bool] = None):
    todos = list(todos_db.values())
    if completed is not None:
        todos = [t for t in todos if t["completed"] == completed]
    return todos

# GET /todos/{todo_id} - получить задачу по ID
@app.get("/todos/{todo_id}", response_model=Todo)
async def get_todo(todo: dict = Depends(get_todo_or_404)):
    return todo

# POST /todos - создать задачу
@app.post("/todos", response_model=Todo, status_code=status.HTTP_201_CREATED)
async def create_todo(todo_data: TodoCreate):
    from datetime import datetime
    todo_id = uuid4()
    todo = {
        **todo_data.dict(),
        "id": todo_id,
        "created_at": datetime.now().isoformat()
    }
    todos_db[todo_id] = todo
    return todo

# PUT /todos/{todo_id} - обновить задачу
@app.put("/todos/{todo_id}", response_model=Todo)
async def update_todo(todo_id: UUID, update_data: TodoUpdate):
    todo = await get_todo_or_404(todo_id)
    
    updated_todo = {**todo}
    for key, value in update_data.dict(exclude_unset=True).items():
        updated_todo[key] = value
    
    todos_db[todo_id] = updated_todo
    return updated_todo

# DELETE /todos/{todo_id} - удалить задачу
@app.delete("/todos/{todo_id}", status_code=status.HTTP_204_NO_CONTENT)
async def delete_todo(todo_id: UUID):
    await get_todo_or_404(todo_id)
    del todos_db[todo_id]
    return None

# GET /health - проверка здоровья сервиса
@app.get("/health")
async def health_check():
    return {"status": "ok", "todos_count": len(todos_db)}
