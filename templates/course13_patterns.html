Урок 1: Введение в паттерны проектирования

Что такое паттерны проектирования?

Паттерны проектирования (Design Patterns) — это проверенные, многократно используемые решения типичных проблем, возникающих при проектировании программного обеспечения.

Это не готовый код, а скорее "чертежи" или "шаблоны", которые можно адаптировать под конкретную задачу. Паттерны помогают писать код, который легче понимать, поддерживать и расширять.

История паттернов:

Классическая книга "Design Patterns: Elements of Reusable Object-Oriented Software" (1994) авторов Erich Gamma, Richard Helm, Ralph Johnson, John Vlissides (так называемая "Банда четырех" или GoF) описала 23 паттерна, которые стали стандартом в мире ООП.

Категории паттернов GoF:

Категория Описание Примеры
Порождающие Описывают механизмы создания объектов, делая систему независимой от способа создания объектов Singleton, Factory, Abstract Factory, Builder, Prototype
Структурные Определяют, как классы и объекты объединяются в более крупные структуры Adapter, Bridge, Composite, Decorator, Facade, Proxy
Поведенческие Описывают алгоритмы и распределение ответственности между объектами Observer, Strategy, Command, State, Template Method, Iterator

Зачем использовать паттерны?

Преимущества использования паттернов:

  • Проверенные решения: Используете опыт тысяч разработчиков до вас
  • Общий язык: "Используем Singleton" понятнее, чем "Создаем класс, который гарантирует, что у нас будет только один экземпляр..."
  • Упрощение поддержки: Код становится предсказуемым и понятным
  • Гибкость и расширяемость: Паттерны помогают строить архитектуру, которую легко модифицировать
  • Уменьшение рисков: Использование проверенных решений снижает вероятность ошибок

Pythonic-way и паттерны:

В Python многие паттерны реализуются проще, чем в других языках, благодаря динамической природе языка. Например:

# Singleton в Python (через метакласс или декоратор)
# Strategy - через функции первого класса
# Observer - через события и сигналы

SOLID принципы — основа паттернов

Паттерны тесно связаны с SOLID принципами ООП:

  • S — Single Responsibility (единая ответственность)
  • O — Open/Closed (открыт для расширения, закрыт для изменения)
  • L — Liskov Substitution (подстановка Барбары Лисков)
  • I — Interface Segregation (разделение интерфейсов)
  • D — Dependency Inversion (инверсия зависимостей)

Паттерны проектирования часто являются конкретными реализациями этих принципов.

Урок 2: Singleton (Одиночка)

Описание паттерна

Singleton — порождающий паттерн, который гарантирует, что у класса есть только один экземпляр, и предоставляет глобальную точку доступа к этому экземпляру.

Когда использовать:

  • Должен быть только один экземпляр класса (например, менеджер конфигурации, пул соединений с БД)
  • Нужна глобальная точка доступа к этому экземпляру
  • Экземпляр должен быть доступен из разных частей программы

Реализация в Python

Способ 1: Через __new__

class DatabaseConnection:
    _instance = None
    
    def __new__(cls):
        if cls._instance is None:
            print("Создаем новое подключение к БД...")
            cls._instance = super().__new__(cls)
            cls._instance.connection = None
        return cls._instance
    
    def connect(self):
        if not self.connection:
            self.connection = "Подключение к базе данных"
            print("Установлено соединение")
        return self.connection
    
    def query(self, sql):
        return f"Выполняем запрос: {sql}"

# Использование
db1 = DatabaseConnection()
db2 = DatabaseConnection()

print(db1 is db2)  # True (один и тот же объект)

db1.connect()
print(db1.query("SELECT * FROM users"))
print(db2.query("SELECT * FROM products"))

Способ 2: Через декоратор

def singleton(cls):
    instances = {}
    
    def get_instance(*args, **kwargs):
        if cls not in instances:
            instances[cls] = cls(*args, **kwargs)
        return instances[cls]
    
    return get_instance

@singleton
class Configuration:
    def __init__(self):
        self.settings = {}
    
    def set(self, key, value):
        self.settings[key] = value
    
    def get(self, key):
        return self.settings.get(key)

# Использование
config1 = Configuration()
config2 = Configuration()

config1.set("theme", "dark")
print(config2.get("theme"))  # dark

Способ 3: Через метакласс

class SingletonMeta(type):
    _instances = {}
    
    def __call__(cls, *args, **kwargs):
        if cls not in cls._instances:
            cls._instances[cls] = super().__call__(*args, **kwargs)
        return cls._instances[cls]

class Logger(metaclass=SingletonMeta):
    def __init__(self):
        self.logs = []
    
    def log(self, message):
        self.logs.append(message)
        print(f"[LOG] {message}")

# Использование
logger1 = Logger()
logger2 = Logger()

logger1.log("Пользователь вошел в систему")
print(len(logger2.logs))  # 1

Плюсы и минусы

Плюсы Минусы
Гарантирует наличие единственного экземпляра Нарушает принцип единой ответственности (управляет и созданием, и работой)
Предоставляет глобальную точку доступа Может создавать проблемы при тестировании (глобальное состояние)
Ленивая инициализация Усложняет многопоточную работу (нужна синхронизация)

Урок 3: Factory Method (Фабричный метод)

Описание паттерна

Factory Method — порождающий паттерн, который определяет интерфейс для создания объекта, но позволяет подклассам решать, какой именно класс инстанцировать.

Когда использовать:

  • Заранее неизвестно, объекты каких типов нужно создавать
  • Класс хочет делегировать создание объектов подклассам
  • Нужно централизовать логику создания объектов

Реализация в Python

from abc import ABC, abstractmethod

# Абстрактный продукт
class Document(ABC):
    @abstractmethod
    def open(self):
        pass
    
    @abstractmethod
    def save(self):
        pass

# Конкретные продукты
class PDFDocument(Document):
    def open(self):
        return "Открываем PDF документ"
    
    def save(self):
        return "Сохраняем PDF документ"

class WordDocument(Document):
    def open(self):
        return "Открываем Word документ"
    
    def save(self):
        return "Сохраняем Word документ"

class ExcelDocument(Document):
    def open(self):
        return "Открываем Excel документ"
    
    def save(self):
        return "Сохраняем Excel документ"

# Абстрактный создатель
class DocumentCreator(ABC):
    @abstractmethod
    def create_document(self):
        """Фабричный метод"""
        pass
    
    def edit_document(self):
        """Операция, использующая фабричный метод"""
        doc = self.create_document()
        print(doc.open())
        print("Редактируем документ...")
        print(doc.save())
        return doc

# Конкретные создатели
class PDFCreator(DocumentCreator):
    def create_document(self):
        return PDFDocument()

class WordCreator(DocumentCreator):
    def create_document(self):
        return WordDocument()

class ExcelCreator(DocumentCreator):
    def create_document(self):
        return ExcelDocument()

# Использование
creators = [PDFCreator(), WordCreator(), ExcelCreator()]

for creator in creators:
    print(f"\n--- {creator.__class__.__name__} ---")
    creator.edit_document()

Pythonic реализация через функции

В Python фабричный метод можно реализовать проще, используя функции первого класса:

class Document:
    pass

class PDFDocument(Document):
    def __str__(self):
        return "PDF документ"

class WordDocument(Document):
    def __str__(self):
        return "Word документ"

# Фабрика через словарь
def create_pdf():
    return PDFDocument()

def create_word():
    return WordDocument()

document_factories = {
    'pdf': create_pdf,
    'word': create_word,
}

def create_document(doc_type):
    factory = document_factories.get(doc_type)
    if factory:
        return factory()
    raise ValueError(f"Unknown document type: {doc_type}")

# Использование
doc = create_document('pdf')
print(doc)  # PDF документ

# Или через параметризованную фабрику
def document_factory(doc_type):
    class_map = {
        'pdf': PDFDocument,
        'word': WordDocument,
    }
    return class_map[doc_type]()

doc2 = document_factory('word')
print(doc2)  # Word документ

Урок 4: Abstract Factory (Абстрактная фабрика)

Описание паттерна

Abstract Factory — порождающий паттерн, который предоставляет интерфейс для создания семейств связанных объектов, не специфицируя их конкретные классы.

Когда использовать:

  • Система должна быть независимой от способа создания объектов
  • Нужно создавать семейства связанных объектов
  • Необходимо гарантировать совместимость создаваемых объектов

from abc import ABC, abstractmethod

# Абстрактные продукты
class Button(ABC):
    @abstractmethod
    def render(self):
        pass

class Checkbox(ABC):
    @abstractmethod
    def render(self):
        pass

# Конкретные продукты для Windows
class WindowsButton(Button):
    def render(self):
        return "Отрисовка кнопки в стиле Windows"

class WindowsCheckbox(Checkbox):
    def render(self):
        return "Отрисовка чекбокса в стиле Windows"

# Конкретные продукты для macOS
class MacButton(Button):
    def render(self):
        return "Отрисовка кнопки в стиле macOS"

class MacCheckbox(Checkbox):
    def render(self):
        return "Отрисовка чекбокса в стиле macOS"

# Абстрактная фабрика
class GUIFactory(ABC):
    @abstractmethod
    def create_button(self) -> Button:
        pass
    
    @abstractmethod
    def create_checkbox(self) -> Checkbox:
        pass

# Конкретные фабрики
class WindowsFactory(GUIFactory):
    def create_button(self) -> Button:
        return WindowsButton()
    
    def create_checkbox(self) -> Checkbox:
        return WindowsCheckbox()

class MacFactory(GUIFactory):
    def create_button(self) -> Button:
        return MacButton()
    
    def create_checkbox(self) -> Checkbox:
        return MacCheckbox()

# Клиентский код
def create_ui(factory: GUIFactory):
    button = factory.create_button()
    checkbox = factory.create_checkbox()
    
    print(button.render())
    print(checkbox.render())

# Использование
os_type = "windows"  # или "mac"

if os_type == "windows":
    factory = WindowsFactory()
else:
    factory = MacFactory()

create_ui(factory)

Урок 5: Observer (Наблюдатель)

Описание паттерна

Observer — поведенческий паттерн, который определяет механизм подписки, позволяющий одним объектам следить за изменениями других объектов.

Когда использовать:

  • Изменения в одном объекте требуют изменений в других объектах
  • Объекты должны быть слабо связаны между собой
  • Нужна система событий и подписчиков

Реализация в Python

from abc import ABC, abstractmethod

# Интерфейс наблюдателя
class Observer(ABC):
    @abstractmethod
    def update(self, message):
        pass

# Конкретные наблюдатели
class EmailNotifier(Observer):
    def __init__(self, email):
        self.email = email
    
    def update(self, message):
        print(f"📧 Отправляем email на {self.email}: {message}")

class SMSNotifier(Observer):
    def __init__(self, phone):
        self.phone = phone
    
    def update(self, message):
        print(f"📱 Отправляем SMS на {self.phone}: {message}")

class PushNotifier(Observer):
    def __init__(self, device_id):
        self.device_id = device_id
    
    def update(self, message):
        print(f"🔔 Отправляем push-уведомление на {self.device_id}: {message}")

# Издатель (субъект)
class NewsPublisher:
    def __init__(self):
        self._observers = []
        self._latest_news = ""
    
    def attach(self, observer):
        self._observers.append(observer)
    
    def detach(self, observer):
        self._observers.remove(observer)
    
    def notify(self):
        for observer in self._observers:
            observer.update(self._latest_news)
    
    def publish_news(self, news):
        self._latest_news = news
        print(f"\n📰 Новость: {news}")
        self.notify()

# Использование
publisher = NewsPublisher()

email_notifier = EmailNotifier("user@example.com")
sms_notifier = SMSNotifier("+7-999-123-45-67")
push_notifier = PushNotifier("iphone-12345")

publisher.attach(email_notifier)
publisher.attach(sms_notifier)
publisher.attach(push_notifier)

publisher.publish_news("Вышел Python 3.13!")

# Можно отписать кого-то
publisher.detach(sms_notifier)
print("\n--- После отписки SMS ---")
publisher.publish_news("Новый курс по паттернам доступен!")

# Использование встроенного модуля для событий
class Event:
    def __init__(self):
        self._handlers = []
    
    def subscribe(self, handler):
        self._handlers.append(handler)
    
    def unsubscribe(self, handler):
        self._handlers.remove(handler)
    
    def fire(self, *args, **kwargs):
        for handler in self._handlers:
            handler(*args, **kwargs)

class User:
    def __init__(self, name):
        self.name = name
        self.on_login = Event()
    
    def login(self):
        print(f"{self.name} вошел в систему")
        self.on_login.fire(self.name)

def send_welcome_email(username):
    print(f"📧 Отправляем приветственное письмо для {username}")

def log_user_activity(username):
    print(f"📝 Логируем вход пользователя {username}")

# Использование
user = User("Анна")
user.on_login.subscribe(send_welcome_email)
user.on_login.subscribe(log_user_activity)

user.login()

Урок 6: Strategy (Стратегия)

Описание паттерна

Strategy — поведенческий паттерн, который определяет семейство алгоритмов, инкапсулирует каждый из них и делает их взаимозаменяемыми.

Когда использовать:

  • Нужно использовать разные варианты алгоритма в одном классе
  • Есть множество похожих классов, отличающихся только поведением
  • Необходимо изолировать логику алгоритмов от клиентского кода

from abc import ABC, abstractmethod
import math

# Интерфейс стратегии
class PaymentStrategy(ABC):
    @abstractmethod
    def pay(self, amount):
        pass

# Конкретные стратегии
class CreditCardPayment(PaymentStrategy):
    def __init__(self, card_number, card_holder):
        self.card_number = card_number
        self.card_holder = card_holder
    
    def pay(self, amount):
        print(f"💳 Оплата {amount} руб. картой {self.card_number[-4:]}")
        return True

class PayPalPayment(PaymentStrategy):
    def __init__(self, email):
        self.email = email
    
    def pay(self, amount):
        print(f"💰 Оплата {amount} руб. через PayPal ({self.email})")
        return True

class CryptoPayment(PaymentStrategy):
    def __init__(self, wallet_address):
        self.wallet_address = wallet_address
    
    def pay(self, amount):
        print(f"🪙 Оплата {amount} руб. криптовалютой ({self.wallet_address[:10]}...)")
        return True

# Контекст
class ShoppingCart:
    def __init__(self):
        self.items = []
        self.payment_strategy = None
    
    def add_item(self, item, price):
        self.items.append({"item": item, "price": price})
    
    def set_payment_strategy(self, strategy: PaymentStrategy):
        self.payment_strategy = strategy
    
    def checkout(self):
        total = sum(item["price"] for item in self.items)
        print(f"\n🛒 Корзина: {len(self.items)} товаров, сумма: {total} руб.")
        
        if self.payment_strategy:
            if self.payment_strategy.pay(total):
                print("✅ Оплата прошла успешно!")
                return True
        else:
            print("❌ Выберите способ оплаты")
        return False

# В Python стратегии могут быть просто функциями
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr

def quick_sort(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[len(arr) // 2]
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    return quick_sort(left) + middle + quick_sort(right)

def python_sort(arr):
    return sorted(arr)

class SortContext:
    def __init__(self, strategy=None):
        self.strategy = strategy or python_sort
    
    def set_strategy(self, strategy):
        self.strategy = strategy
    
    def sort(self, data):
        return self.strategy(data.copy())

# Использование
data = [64, 34, 25, 12, 22, 11, 90]

context = SortContext(python_sort)
print("Python sort:", context.sort(data))

context.set_strategy(bubble_sort)
print("Bubble sort:", context.sort(data))

context.set_strategy(quick_sort)
print("Quick sort:", context.sort(data))

Урок 7: Decorator (Декоратор)

Описание паттерна

Decorator — структурный паттерн, который позволяет динамически добавлять объектам новую функциональность, оборачивая их в полезные "обертки".

Когда использовать:

  • Нужно добавлять обязанности объектам динамически и прозрачно
  • Наследование нецелесообразно или невозможно
  • Нужно избежать раздувания иерархии классов

from abc import ABC, abstractmethod

# Компонент
class Coffee(ABC):
    @abstractmethod
    def get_cost(self):
        pass
    
    @abstractmethod
    def get_description(self):
        pass

# Конкретный компонент
class SimpleCoffee(Coffee):
    def get_cost(self):
        return 100
    
    def get_description(self):
        return "Простой кофе"

# Абстрактный декоратор
class CoffeeDecorator(Coffee):
    def __init__(self, coffee: Coffee):
        self._coffee = coffee
    
    def get_cost(self):
        return self._coffee.get_cost()
    
    def get_description(self):
        return self._coffee.get_description()

# Конкретные декораторы
class MilkDecorator(CoffeeDecorator):
    def get_cost(self):
        return self._coffee.get_cost() + 30
    
    def get_description(self):
        return self._coffee.get_description() + ", с молоком"

class SugarDecorator(CoffeeDecorator):
    def get_cost(self):
        return self._coffee.get_cost() + 10
    
    def get_description(self):
        return self._coffee.get_description() + ", с сахаром"

class ChocolateDecorator(CoffeeDecorator):
    def get_cost(self):
        return self._coffee.get_cost() + 50
    
    def get_description(self):
        return self._coffee.get_description() + ", с шоколадом"

class WhippedCreamDecorator(CoffeeDecorator):
    def get_cost(self):
        return self._coffee.get_cost() + 40
    
    def get_description(self):
        return self._coffee.get_description() + ", со взбитыми сливками"

# Использование

В Python есть встроенная поддержка декораторов на уровне синтаксиса:

import time
import functools

# Декоратор для логирования
def log(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print(f"Вызов функции: {func.__name__}")
        print(f"Аргументы: args={args}, kwargs={kwargs}")
        result = func(*args, **kwargs)
        print(f"Результат: {result}")
        return result
    return wrapper

# Декоратор для замера времени
def timer(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f"{func.__name__} выполнилась за {end - start:.4f} сек")
        return result
    return wrapper

# Декоратор для кэширования
def cache(func):
    _cache = {}
    
    @functools.wraps(func)
    def wrapper(*args):
        if args in _cache:
            print(f"Возвращаем из кэша: {args}")
            return _cache[args]
        result = func(*args)
        _cache[args] = result
        return result
    return wrapper

# Декоратор для проверки типов
def type_check(expected_types):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args):
            for arg, expected in zip(args, expected_types):
                if not isinstance(arg, expected):
                    raise TypeError(f"Ожидался {expected}, получен {type(arg)}")
            return func(*args)
        return wrapper
    return decorator

# Использование нескольких декораторов
@timer
@log
def slow_function(n):
    time.sleep(n)
    return f"Завершено через {n} сек"

@cache
def expensive_computation(x, y):
    print("Выполняем сложные вычисления...")
    time.sleep(2)
    return x ** y

@type_check((int, int))
def multiply(a, b):
    return a * b

# Применение
slow_function(1)
expensive_computation(2, 10)
expensive_computation(2, 10)  # Из кэша
print(multiply(5, 3))
